## 0. 사전 요구사항

### 개발 PC (빌드/개발)

| 항목 | 요구 사양 | 비고 |
|------|---------|------|
| OS | Windows 10/11 64-bit | Mac/Linux 미지원 (PyInstaller spec 가 Windows 전용) |
| CPU | 4코어 이상 | 빌드 시간에 영향 |
| RAM | 16 GB 이상 권장 | PyInstaller analysis 단계가 메모리 사용 큼 |
| GPU | **NVIDIA RTX 3060+** (VRAM 6 GB+) | 백엔드 동작 검증에 필요 (CUDA 12.x) |
| 디스크 | 50 GB 여유 | 모델 + node_modules + 빌드 산출물 + setup.exe |
| 인터넷 | 안정 회선 (~14 GB HF 다운로드) | 첫 setup 시 모델 받음 |

### 사용자 PC (배포 대상)

| 항목 | 요구 사양 | 사용자가 따로 설치? |
|------|---------|---|
| OS | Windows 10/11 64-bit | — |
| GPU | NVIDIA RTX 3060+ | ✅ 드라이버 CUDA 12.x |
| 디스크 | **35 GB 여유** | ✅ 앱 본체 3.5 GB + VLM ~14 GB + 사본 ~14 GB |
| Python / Node | 없어도 됨 | ❌ PyInstaller / Electron 이 내장 |
| 관리자 권한 | 없어도 됨 | ❌ per-user 설치 (`%LOCALAPPDATA%`) |
| 인터넷 (첫 실행만) | 필요 | ✅ VLM 14 GB 다운로드 (~30~60분) |

## 1. 도구 설치 (개발 PC)

다음 도구가 모두 설치되어 있어야 빌드가 가능합니다. 각각 단독 설치.

```powershell
# 1. Node.js 20+ (LTS) — nodejs.org 에서 다운로드
node --version    # v20+ 확인

# 2. Miniconda or Anaconda — conda.io 에서 다운로드
conda --version

# 3. NVIDIA 드라이버 (CUDA 12.x 지원 버전)
nvidia-smi        # CUDA Version: 12.x 표시 확인

# 4. Git
git --version

# 5. Inno Setup 6.x — 설치본 컴파일러
winget install JRSoftware.InnoSetup
# 또는 https://jrsoftware.org/isdl.php 에서 다운로드
# 설치 후 확인:
# %LOCALAPPDATA%\Programs\Inno Setup 6\ISCC.exe
```

> Visual Studio Build Tools 등은 불필요 — PyInstaller / Electron 이 알아서 처리.

## 2. 저장소 클론 + 초기 setup

```powershell
# 1. 클론
git clone https://lab.ssafy.com/s14-final/S14P31S205.git
cd S14P31S205

# 2. 원클릭 설치 (약 20~40분, 약 10 GB 다운로드)
npm run setup
```

`npm run setup` 이 자동으로 처리하는 것:

| # | 단계 | 산출물 |
|---|------|------|
| 1 | 루트 npm 패키지 설치 | `node_modules/` |
| 2 | 프론트엔드 npm 패키지 설치 + Silero VAD ONNX 복사 | `frontend/node_modules/`, `frontend/public/silero_vad_*.onnx` |
| 3 | `.env.example` → `.env` 복사 | `.env` |
| 4 | conda `aunion` 환경 생성 (Python 3.11) | conda env |
| 5 | Python 패키지 설치 (CUDA 12.6 PyTorch 포함) | `aunion` 환경 의존성 |
| 6 | AI 모델 다운로드 (Whisper / NLLB / Surya) + CT2 변환 | `models/whisper-large-v3-turbo-ct2-int8/`, `models/nllb-200-distilled-600M-ct2/` |
| 7 | 필수 파일 존재 검증 | (검증만) |

> **`npm run setup` 이 중간에 실패하면**: 트러블슈팅 섹션 참고. 대부분 conda env 가 dirty 한 경우라 `conda env remove -n aunion` 후 재실행으로 해결.

## 4. 설치본 빌드 (master/dev 브랜치 → setup.exe)

### 한 줄로

```powershell
npm run build:installer
```

내부 4단계 (순차 실행, 총 ~50분):

| # | 단계 | 명령 | 산출물 | 소요 |
|---|------|------|------|------|
| 1 | Vite 프론트 빌드 | `npm run build --prefix frontend` | `frontend/dist/` | ~1분 |
| 2 | PyInstaller 백엔드 패키징 | `pyinstaller backend/aunion.spec` | `backend/dist/aunion_backend/` (5 GB+) | ~10~15분 |
| 3 | electron-builder Electron 패키징 | `electron-builder --config electron-builder.json` | `setup/win-unpacked/` (7 GB) | ~5~7분 |
| 4 | Inno Setup 압축 → 단일 .exe | `node scripts/run-iscc.js installer.iss` | `setup/Aunion-AI-Setup-{ver}.exe` (3.4 GB) | ~20~30분 |

### 안전 가드 (자동)

- PyInstaller 진입 시 `frontend/dist/index.html` 존재 검사 → 없으면 빌드 중단 ("먼저 `npm run build --prefix frontend` 실행하세요"). 옛 stale dist 가 setup.exe 에 박히는 사고 방지.

### 빌드 전 깨끗이 시작하기 (이전 산출물 잔재 제거)

```powershell
# 실행 중인 앱 종료
Get-Process | Where-Object { $_.ProcessName -match '^(aunion|Aunion)' } | Stop-Process -Force

# 이전 빌드 산출물 삭제
Remove-Item -Recurse -Force frontend\dist, backend\dist, backend\build, setup -ErrorAction SilentlyContinue

# (선택) 이전 설치본 제거 — 사용자 데이터는 `%LOCALAPPDATA%\Aunion AI\` 에 별도 보존됨
Remove-Item -Recurse -Force "$env:LOCALAPPDATA\Programs\Aunion AI" -ErrorAction SilentlyContinue
```

## 5. 설치본 배포 / 설치

### 배포할 파일

```
setup/Aunion-AI-Setup-{version}.exe   ← 이 파일 1개만 전달
```

크기 ~3.4 GB. 클라우드 스토리지 / 외장 SSD / 사내 파일서버 등으로 전달.

### 설치 방법 (사용자 PC)

1. `Aunion-AI-Setup-{version}.exe` 더블클릭
2. 마법사 진행 (UAC 안 뜸 — per-user 설치)
3. 설치 완료 → 자동 실행
4. 첫 실행 마법사:
   - 디스크 35 GB 여유 확인
   - VLM 미캐시면 "다운로드 시작" 클릭 → ~14 GB / 30~60분
   - 모델 검증 → Complete → `/lecturer` 자동 진입

### 설치 위치

```
%LOCALAPPDATA%\Aunion AI\
├─ Aunion AI.exe                # 앱 본체
├─ resources\backend\           # PyInstaller 백엔드 (재설치 시 덮어쓰기)
├─ cache\huggingface\           # VLM 모델 (재설치 시 보존)
├─ uploads\                     # 강의자료 (재설치 시 보존)
├─ transcripts\                 # 강의 자막 (재설치 시 보존)
├─ logs\
└─ error_log.txt
```

`resources/backend/` 만 인스톨러가 덮어쓰는 영역. 그 외는 사용자 데이터 — 재설치/업그레이드해도 보존.

## 6. 동작 검증 시나리오

설치 직후 다음 시나리오로 핵심 기능 4개 확인:

### ① 강의자 흐름

1. 시작메뉴에서 Aunion AI 실행
2. 강의자료 PDF 업로드 → 슬라이드 번역 진행률 → 완료
3. "초대 링크 복사" → 학생용 URL 획득
4. 강의 시작 → 마이크 켜고 발화 → 자막 자동 생성 확인

### ② 학생 흐름 (다른 PC 또는 같은 PC 의 다른 브라우저)

1. 학생용 URL 접속 → 이름 입력 → 음성 옵션 **"원본 (Original)"** 선택
2. 입장 → 강의자 화면 + 자막 표시
3. 강의자 발화 → 학생에게 들리는지 확인 ✅
4. 음성 옵션 "영어 (English)" 토글 → piper TTS 음성 재생 확인 ✅

### ③ 일시정지 ↔ 재개

1. 강의자가 일시정지 → 학생 측 원본 음성 즉시 mute ✅
2. 강의자가 재개 → 음성 자동 복귀 ✅

### ④ 재설치 시 사용자 데이터 보존

1. 설치본 1차 설치 → 강의자료 업로드 + VLM 다운로드 완료
2. 같은 setup.exe 다시 실행 (또는 새 버전)
3. 설치 완료 후 앱 실행 → 이전 업로드 자료 + VLM 캐시 그대로 ✅

## 7. 트러블슈팅 (자주 발생 순)

### `npm run setup` 도중 실패

원인 대부분: conda env 가 부분 생성 상태로 종료됨.

```powershell
conda env remove -n aunion -y
npm run setup
```

### `pyinstaller` 가 frontend/dist 없다고 빌드 중단

빌드 가드 의도된 동작.

```powershell
npm run build --prefix frontend
# 그 다음 다시
npm run build:installer
```

### 설치본이 마이크 / 카메라 못 잡음

`http://127.0.0.1:48000` 으로 frontend 로딩되어야 secure context 인정.
백엔드가 늦게 뜨면 빈 창이 잠시 보일 수 있음 — 정상.

여전히 안 되면: `%LOCALAPPDATA%\Aunion AI\error_log.txt` 확인.

### `[WinError 448] 신뢰할 수 없는 탑재 지점`

옛 admin 빌드 cache 의 symlink 잔재. fix:

```powershell
Remove-Item -Recurse -Force "$env:LOCALAPPDATA\Aunion AI\cache"
# 앱 재시작 → 자동 재다운로드
```

### Inno Setup 컴파일 실패 ("ISCC.exe 못 찾음")

`scripts/run-iscc.js` 가 자동 탐색하지만 못 찾을 수 있음. 수동 호출:

```powershell
& "$env:LOCALAPPDATA\Programs\Inno Setup 6\ISCC.exe" installer.iss
```

### 빌드 중 PyInstaller 가 `[hook-xxx]` 단계에서 멈춤

메모리 부족 또는 안티바이러스 간섭. 임시로 안티바이러스 꺼보고, 그래도 안 되면 `backend/build/`, `backend/dist/` 지우고 재시도.

### 강사 발화가 학생에게 안 들림

이미 픽스됨 (재협상 트리거). 그래도 발생 시 강사 콘솔 F12 에서:

```
[Lecturer] 초기 replaceTrack → audio=null|<id>
[Lecturer] swap → audio=<id> renegotiate=true
[SDP] 재협상 offer 전송 →
```

로그 패턴이 보이는지 확인. 보이는데 학생 무음이면 학생 콘솔의 `[Track] audio onunmute` 와 `[Stats] inbound audio packets=` 확인.

### 더 자세한 트러블슈팅

[Electron-설치파일-생성방법.md](Electron-설치파일-생성방법.md#자주-막히는-지점-트러블슈팅) 의 "자주 막히는 지점" 섹션에 빌드/설치 관련 추가 케이스 정리되어 있습니다.

## 8. 디렉터리 구조 (저장소)

```
S14P31S205/
├─ backend/                  # Python FastAPI 백엔드
│   ├─ app/                  # 라우터 / 서비스
│   ├─ aunion.spec           # PyInstaller spec (빌드 가드 포함)
│   └─ run.py                # 백엔드 엔트리포인트
├─ frontend/                 # Vite + React + Electron
│   ├─ electron/             # main.cjs / preload.cjs / 아이콘
│   ├─ src/                  # React 소스
│   ├─ public/               # 정적 자원 (VAD ONNX 등)
│   └─ package.json
├─ turn-server/              # node-turn 로컬 TURN 서버
├─ models/                   # AI 모델 (gitignore, npm run setup 으로 받음)
├─ installer-assets/         # Inno Setup 마법사 비주얼 자산
├─ scripts/                  # setup.js / ensure-env.js / run-iscc.js 등
├─ docs/                     # 본 문서 포함 모든 .md
├─ electron-builder.json
├─ installer.iss             # Inno Setup 설정
├─ package.json              # 루트 — 빌드 스크립트 (build:installer 등)
├─ .env.example
└─ .gitignore
```

## 9. 버전 갱신

1. `package.json` 의 `version` 올림 (예: `0.1.0` → `0.2.0`)
2. `installer.iss` 의 `MyAppVersion` 동일하게 수정
3. `npm run build:installer` 재실행
4. 새 `setup/Aunion-AI-Setup-0.2.0.exe` 배포
5. 사용자는 새 setup.exe 실행 → 자동 덮어쓰기 설치 (사용자 데이터는 자동 보존)
