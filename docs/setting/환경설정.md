# 환경설정 가이드

프로젝트 루트의 `.env` 파일로 AI 모델 및 디바이스를 설정합니다.

---

## .env 전체 항목

```env
# HuggingFace 토큰 (비공개 모델 사용 시 필요, 공개 모델은 없어도 됨)
HF_TOKEN=hf_xxxxxxxxxxxx

# AI 모델명 (HuggingFace 저장소 ID)
ASR_MODEL=ghost613/faster-whisper-large-v3-turbo-korean
NMT_MODEL=Helsinki-NLP/opus-mt-ko-en
TTS_MODEL=facebook/mms-tts-eng
OCR_MODEL=rapidocr

# 디바이스 설정 (cpu 또는 cuda)
ASR_DEVICE=cuda
NMT_DEVICE=cuda
TTS_DEVICE=cuda
OCR_DEVICE=cuda
```

---

## 디바이스 설정

| 모델 | GPU 지원 | 권장 설정 |
|------|----------|-----------|
| ASR (음성인식) | ✅ | `cuda` (GPU 있을 때) |
| NMT (번역) | ✅ | `cuda` (GPU 있을 때) |
| TTS (음성합성) | ✅ | `cuda` (GPU 있을 때) |
| OCR (문자인식) | ✅ | `cuda` (GPU 있을 때) |

---

## OCR 모델 변경

`OCR_MODEL`은 두 가지 방식을 지원합니다.

| 값 | 설명 |
|----|------|
| `rapidocr` (기본값) | 경량 ONNX 기반, 한국어/영어 지원, CPU 최적화 |
| HuggingFace 모델 ID | transformers 기반 (예: `microsoft/trocr-base-printed`) |

```env
# 기본값
OCR_MODEL=rapidocr

# HuggingFace 모델로 교체 시
OCR_MODEL=microsoft/trocr-base-printed
```

> HuggingFace 모델 사용 시 bbox(위치 정보)가 반환되지 않아 슬라이드 오버레이 기능이 제한됩니다.

### 자동 폴백 동작

- `cuda` 설정 + CUDA 미설치 환경 → 자동으로 CPU로 전환 (오류 없음)
- GPU 없는 환경에서도 .env를 그대로 유지해도 됩니다.

---

## AI 모델 변경

`.env`에서 모델명만 바꾸면 됩니다. 재시작 시 자동으로 새 모델을 다운로드합니다.

```env
# 예: 다른 ASR 모델로 변경
ASR_MODEL=openai/whisper-large-v3
```

모델 캐시 위치: `%LOCALAPPDATA%\Aunion AI\cache\huggingface`

---

## 배포판(exe)에서 설정 변경

설치 경로 내 `.env` 파일을 직접 수정합니다:

```
<설치경로>\resources\backend\.env
```

기본 설치 경로 예시:
```
C:\Users\USERNAME\AppData\Local\Programs\Aunion AI\resources\backend\.env
```
