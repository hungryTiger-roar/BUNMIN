# 환경설정 가이드

프로젝트 루트(`.env`)의 단일 파일로 백엔드·프론트엔드 모든 설정을 관리합니다.

> **왜 .env로 관리하나요?**  
> 모델명·디바이스가 코드에 하드코딩되어 있으면, 모델을 교체할 때마다 코드를 수정하고 재배포해야 합니다.  
> `.env`로 분리하면 파일 한 줄만 바꾸고 재시작하는 것으로 적용이 끝납니다.  
> 팀원마다 GPU/CPU 환경이 다를 때도 각자 `.env`만 수정하면 되므로 코드 충돌이 없습니다.

---

## .env 전체 항목

```env
# HuggingFace 토큰 (공개 모델만 사용하므로 선택사항)
HF_TOKEN=

# OpenAI API 키 (선택사항 — 슬라이드 번역 시 용어집 자동 생성)
OPENAI_API_KEY=

ASR_MODEL=models/whisper-large-v3-turbo-ct2-int8
NMT_ASR_MODEL=facebook/nllb-200-distilled-600M
OCR_MODEL=surya

ASR_DEVICE=cuda
NMT_ASR_DEVICE=cuda
OCR_DEVICE=cpu

# 서버 시작 시 모델 로딩 여부
# false: 실시간 번역 전체 기능 (ASR + NMT + OCR 로드, 기본값)
# true:  슬라이드 번역 전용 모드
SKIP_STARTUP_MODELS=false

# VLM 설정 (슬라이드 번역)
VLM_BASE_MODEL=models/qwen2.5-vl-7b-instruct
VLM_DEVICE=cuda
VLM_MAX_GPU_MEMORY=5.5GB
VLM_USE_4BIT=true
```

### ASR_DEVICE / NMT_ASR_DEVICE / OCR_DEVICE

| 항목 | 내용 |
|------|------|
| 사용 이유 | 각 모델을 GPU(cuda)와 CPU(cpu) 중 어디서 실행할지 지정합니다. |
| 사용 위치 | `backend/app/config.py` — 런타임에 CUDA 설치 여부를 체크해 디바이스 결정 |
| 자동 폴백 | `cuda`로 설정해도 CUDA 미설치 환경이면 자동으로 `cpu`로 전환됩니다. |

| 값 | 설명 |
|----|------|
| `cuda` | NVIDIA GPU 사용. CUDA 및 드라이버 설치 필요 |
| `cpu` | CPU 사용. GPU 없는 환경에서 사용 |

> ASR/NMT 는 실시간 번역에 GPU 가속이 중요하므로 `cuda` 를 권장합니다 (각 ~1GB VRAM, 6GB GPU 에 동시 적재 가능).  
> OCR(Surya)는 GPU 가속을 지원하며, `cuda` 를 권장합니다.

| 항목 | 내용 |
|------|------|
| 사용 이유 | Vite 개발 서버가 백엔드 API를 프록시할 주소를 지정합니다. |
| 사용 위치 | `frontend/vite.config.ts` — `/api`, `/slides`, `/health`, `/network`, `/transcripts`, `/ws` 경로를 이 주소로 프록시 |
| 기본값 | 설정 없으면 `http://localhost:48000`으로 동작 |


> **주의**: 이 변수는 Vite 빌드/개발 서버 실행 시에만 읽히는 서버 사이드 변수입니다.  
> `VITE_` 접두사가 없으므로 클라이언트 번들에 노출되지 않습니다.  
> Electron 배포판은 항상 `127.0.0.1:48000`을 직접 참조하므로 이 설정이 적용되지 않습니다.

### VITE_SYNC_DELAY_MS (선택)

| 항목 | 내용 |
|------|------|
| 사용 이유 | 수강자측 wall-clock delay-buffer 의 **초기 / 기준 lag** (ms). 강사 wall-clock 기준 모든 출력 (원본 음성 / TTS / 그림 / 자막) 을 이 시간만큼 지연시켜 정렬합니다. 실제 적용 lag 은 실측 처리시간 따라 적응형 (`currentDelay`) 으로 가변. |
| 사용 위치 | `frontend/src/pages/Student.tsx` — `useDelayBufferPlayer` 및 원본 음성 DelayNode 의 공통 초기 lag |
| 기본값 | `10000` (10초) — 첫 발화 전 사용되는 보수적 기준값. 이후 `recentP90(ASR+MT+TTS 처리시간) + jitter 마진` 으로 동적 조정 |
| 적응형 범위 | `DELAY_MIN_MS=2000` (2초) ~ `DELAY_MAX_MS=20000` (20초). silence 구간에선 자연히 최솟값으로 수렴 |
| 변경 시 영향 | 값을 줄이면 첫 발화 라이브감 ↑ 이지만 첫 발화 처리시간이 길면 desync 위험. 늘리면 첫 발화 안정성 ↑ 이지만 약간 더 lag. 둘째 발화부턴 어차피 적응형이라 영향 작음 |

```env
# 빠른 네트워크 환경에서 라이브감 강조
VITE_SYNC_DELAY_MS=8000
```

> **주의**: `VITE_` 접두사 변수는 Vite **빌드 시점**에 `import.meta.env` 로 클라이언트 번들에 inject 됩니다.  
> 런타임에 바꿀 수 없으므로 변경 후 프론트 재빌드 (`npm run build`) 필요.  
> 강사측 송출에도 같은 lag 을 가정하므로 강사/수강자 빌드의 값이 다르면 sync 가 깨집니다 — 동일 빌드를 양쪽에서 사용하세요.

## WebRTC TURN 서버 (로컬)

원본 음성 (WebRTC) 이 P2P 차단 환경 (SSAFY 캠퍼스, AP client isolation 등) 에서 끊기는 문제 해결용. 강사 노트북에서 `node-turn` 이 같이 실행되며, 모든 수강자가 같은 LAN 의 이 TURN 서버를 거쳐 relay.

| 항목 | 내용 |
|------|------|
| 실행 위치 | `turn-server/index.js` — `npm run dev` / `electron:dev` 가 concurrently 로 자동 가동. Electron 운영판은 `main.cjs` 의 `startTurnServer()` 가 기동 |
| 포트 | `47878` (UDP + TCP 동시) — TURN 표준 3478 은 Teams/Zoom 등이 자주 점유해 충돌 회피용 임의 포트로 변경 |
| 인증 | long-term, user: `aunion` / credential: `aunion-secret` (코드 하드코딩) |
| iceServers | 프론트가 `getLanIp()` (`/network/info` 응답 캐시) 로 강사 PC LAN IP 확보 → `turn:<lan_ip>:47878` UDP + TCP 두 transport 등록 |
| 의존성 | `turn-server/node_modules/node-turn` — `npm run setup` 이 자동 설치 |

### 주의

- **Windows 방화벽**: 처음 `npm run dev` 실행 시 Node.js 외부 접근 권한 팝업 → "개인 네트워크 허용" 체크 필요.
- **포트 충돌**: `47878` 이 점유돼 있으면 TURN 가동 실패 → `netstat -ano | findstr 47878` 로 확인 후 종료. (Teams/Zoom 등은 표준 3478 만 점유하므로 47878 충돌 가능성은 매우 낮음.)
- **강사 접속 URL**: 강사도 학생과 동일한 LAN IP 의 URL (`http://192.168.x.x:43000`) 로 접속해야 함 (dev 모드 기준). `localhost` 로 접속 시 학생 측에서 강사 TURN 도달 불가.
- **운영 배포 단계**: 로컬 TURN 은 같은 LAN 한정 → 일반 사용자 배포 시 클라우드 TURN (AWS coturn 등) 필요. 그 시점에 `iceServers` 를 환경변수 / backend 응답으로 동적 구성하도록 리팩토링 필요.

## Electron 운영판 다운로드 / 외부 링크

`Lecturer.tsx`, `Student.tsx` 의 다운로드 버튼은 `window.open(url, '_blank')` 으로 호출됩니다. Electron 은 기본적으로 빈 BrowserWindow 를 생성해 흰 창이 떠 보였습니다.

`frontend/electron/main.cjs` 의 `setWindowOpenHandler` 가 다음과 같이 처리:

| URL 패턴 | 처리 |
|----------|------|
| `/slides/download/...`, `/transcripts/...` | `webContents.downloadURL(url)` — Chrome 식 다운로드 매니저 |
| 기타 `http(s)://...` | `shell.openExternal(url)` — 시스템 기본 브라우저 |
| 그 외 | `action: 'deny'` 로 차단 |

→ Renderer 코드 변경 없이 Electron 운영판에서만 다른 동작. 웹 브라우저에선 `_blank` 새 탭이 그대로 열립니다.

## 배포판(exe)에서 설정 변경

설치 경로 내 `.env` 파일을 직접 수정합니다.

```
<설치경로>\resources\backend\.env
```

기본 설치 경로 예시:
```
C:\Users\USERNAME\AppData\Local\Programs\Aunion AI\resources\backend\.env
```
