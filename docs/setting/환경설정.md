# 환경설정 가이드

프로젝트 루트(`.env`)의 단일 파일로 백엔드·프론트엔드 모든 설정을 관리합니다.

> **왜 .env로 관리하나요?**
> 모델명·디바이스가 코드에 하드코딩되어 있으면, 모델을 교체할 때마다 코드를 수정하고 재배포해야 합니다.
> `.env`로 분리하면 파일 한 줄만 바꾸고 재시작하는 것으로 적용이 끝납니다.
> 팀원마다 GPU/CPU 환경이 다를 때도 각자 `.env`만 수정하면 되므로 코드 충돌이 없습니다.

---

## .env 전체 항목

```env
HF_TOKEN=hf_xxxxxxxxxxxx

ASR_MODEL=CohereLabs/cohere-transcribe-03-2026
NMT_MODEL=tencent/HY-MT1.5-1.8B
TTS_MODEL=piper
OCR_MODEL=rapidocr

ASR_DEVICE=cuda
NMT_DEVICE=cuda
TTS_DEVICE=cpu
OCR_DEVICE=cpu
```

---

## 항목별 사용 이유 및 설명

### HF_TOKEN

| 항목 | 내용 |
|------|------|
| 사용 이유 | HuggingFace gated 모델 다운로드 시 인증에 사용됩니다. |
| 필수 여부 | 기본 ASR 모델(`CohereLabs/cohere-transcribe-03-2026`)은 gated 모델이므로 **HF_TOKEN 필수**입니다. HuggingFace에서 해당 모델 접근 승인도 받아야 합니다. |
| 발급 방법 | https://huggingface.co/settings/tokens 에서 발급 |
| 주의 | 인증 정보이므로 `.gitignore`에 등록되어 있습니다. 절대 커밋하지 마세요. |

---

### ASR_MODEL

| 항목 | 내용 |
|------|------|
| 사용 이유 | 강의자의 한국어 음성을 텍스트로 변환하는 ASR 모델을 지정합니다. |
| 기본값 | `CohereLabs/cohere-transcribe-03-2026` — Cohere 한국어 음성인식 모델 (GPU: bfloat16, CPU: float32) |
| 사용 위치 | `backend/app/services/asr_service.py` — transformers AutoModelForSpeechSeq2Seq (trust_remote_code=True) |
| 변경 시 | HuggingFace ASR 호환 모델로 교체 가능 |

```env
# 예: OpenAI 공식 whisper 모델로 변경
ASR_MODEL=openai/whisper-large-v3
```

---

### NMT_MODEL

| 항목 | 내용 |
|------|------|
| 사용 이유 | ASR로 인식된 한국어 텍스트를 영어로 번역하는 NMT 모델을 지정합니다. |
| 기본값 | `tencent/HY-MT1.5-1.8B` — Tencent 한→영 번역 causal LM (GPU: float16, CPU: float32) |
| 사용 위치 | `backend/app/services/nmt_service.py` — AutoModelForCausalLM + chat template |
| 변경 시 | chat template 지원 HuggingFace causal LM으로 교체 가능 |

---

### TTS_MODEL

| 항목 | 내용 |
|------|------|
| 사용 이유 | 번역된 영어 텍스트를 음성으로 합성하는 TTS 모델을 지정합니다. |
| 기본값 | `piper` — Piper TTS (en_US-lessac-medium.onnx), 로컬 ONNX 모델 자동 다운로드 |
| 사용 위치 | `backend/app/services/tts_service.py` — piper-tts 패키지 |
| 변경 시 | 현재 Piper 고정 (다른 모델로 변경 시 서비스 코드 수정 필요) |

---

### OCR_MODEL

| 항목 | 내용 |
|------|------|
| 사용 이유 | 슬라이드 이미지에서 텍스트를 추출하는 OCR 모델을 지정합니다. |
| 기본값 | `rapidocr` — 경량 ONNX 기반, 한국어/영어 지원, CPU 최적화 |
| 사용 위치 | `backend/app/services/ocr_service.py` — 값에 따라 RapidOCR 또는 HuggingFace pipeline 로드 |
| 변경 시 | HuggingFace 모델 ID로 교체 가능. 단, bbox(위치 정보)가 반환되지 않아 슬라이드 오버레이 기능이 제한됩니다. |

```env
# 기본값 (권장)
OCR_MODEL=rapidocr

# HuggingFace 모델로 교체 시
OCR_MODEL=microsoft/trocr-base-printed
```

---

### ASR_DEVICE / NMT_DEVICE / TTS_DEVICE / OCR_DEVICE

| 항목 | 내용 |
|------|------|
| 사용 이유 | 각 모델을 GPU(cuda)와 CPU(cpu) 중 어디서 실행할지 지정합니다. |
| 사용 위치 | `backend/app/config.py` — 런타임에 CUDA 설치 여부를 체크해 디바이스 결정 |
| 자동 폴백 | `cuda`로 설정해도 CUDA 미설치 환경이면 자동으로 `cpu`로 전환됩니다. |

| 값 | 설명 |
|----|------|
| `cuda` | NVIDIA GPU 사용. CUDA 및 드라이버 설치 필요 |
| `cpu` | CPU 사용. GPU 없는 환경에서 사용 |

> ASR·NMT는 GPU 가속이 실시간 처리에 중요하므로 `cuda`를 권장합니다. TTS(Piper)와 OCR(RapidOCR)은 ONNX CPU 전용 모델이므로 반드시 `cpu`로 설정하세요.

---

### VITE_API_URL (선택)

| 항목 | 내용 |
|------|------|
| 사용 이유 | 프론트엔드가 백엔드 API를 호출할 주소를 지정합니다. |
| 사용 위치 | `frontend/src/hooks/useWebSocket.ts`, `frontend/src/components/lecturer/SlideViewer.tsx` |
| 기본값 | 없어도 `http://localhost:8000`으로 자동 동작 (fallback 처리됨) |
| 원격 서버 연결 시 | `.env`에 추가 후 서버 IP로 변경 |

```env
# 원격 서버 연결 시만 추가
VITE_API_URL=http://192.168.0.10:8000
```

> **VITE_ 접두사가 붙는 이유**: Vite는 보안상 `VITE_`로 시작하는 변수만 클라이언트 번들에 노출합니다.

## AI 모델 변경 방법

`.env`에서 모델명만 바꾸고 재시작하면 자동으로 새 모델을 다운로드합니다.

```env
# 예: ASR 모델 변경
ASR_MODEL=openai/whisper-large-v3
```

모델 캐시 위치:
- 개발환경: `backend/cache/huggingface`
- 배포판(exe): `%LOCALAPPDATA%\Aunion AI\cache\huggingface`

---

## 배포판(exe)에서 설정 변경

설치 경로 내 `.env` 파일을 직접 수정합니다.

```
<설치경로>\resources\backend\.env
```

기본 설치 경로 예시:
```
C:\Users\USERNAME\AppData\Local\Programs\Aunion AI\resources\backend\.env
```
