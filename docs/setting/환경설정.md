# 환경설정 가이드

프로젝트 루트(`.env`)의 단일 파일로 백엔드·프론트엔드 모든 설정을 관리합니다.

> **왜 .env로 관리하나요?**
> 모델명·디바이스가 코드에 하드코딩되어 있으면, 모델을 교체할 때마다 코드를 수정하고 재배포해야 합니다.
> `.env`로 분리하면 파일 한 줄만 바꾸고 재시작하는 것으로 적용이 끝납니다.
> 팀원마다 GPU/CPU 환경이 다를 때도 각자 `.env`만 수정하면 되므로 코드 충돌이 없습니다.

---

## .env 전체 항목

```env
# 필수
ASR_MODEL=ghost613/faster-whisper-large-v3-turbo-korean
NMT_ASR_MODEL=Helsinki-NLP/opus-mt-ko-en
TTS_MODEL=piper
OCR_MODEL=rapidocr

ASR_DEVICE=cuda
NMT_ASR_DEVICE=cpu
TTS_DEVICE=cpu
OCR_DEVICE=cpu

# 선택 (HuggingFace gated 모델 사용 시)
HF_TOKEN=hf_xxxxxxxxxxxx

# 선택 (개발 환경, 원격 백엔드 연결 시)
BACKEND_URL=http://192.168.0.10:8000

# 선택 (VLM GPU 설정)
VLM_MAX_GPU_MEMORY=6GB
VLM_USE_4BIT=true

# 선택 (슬라이드 번역 전용 모드)
SKIP_STARTUP_MODELS=false
```

---

## 항목별 사용 이유 및 설명

### HF_TOKEN

| 항목 | 내용 |
|------|------|
| 사용 이유 | HuggingFace gated 모델 다운로드 시 인증에 사용됩니다. |
| 필수 여부 | 기본 모델(`ghost613/faster-whisper-large-v3-turbo-korean`, `Helsinki-NLP/opus-mt-ko-en`)은 공개 모델이므로 **기본 설정에서는 불필요**합니다. gated 모델로 교체 시에만 필요합니다. |
| 발급 방법 | https://huggingface.co/settings/tokens 에서 발급 |
| 주의 | 인증 정보이므로 `.gitignore`에 등록되어 있습니다. 절대 커밋하지 마세요. |

---

### ASR_MODEL

| 항목 | 내용 |
|------|------|
| 사용 이유 | 강의자의 한국어 음성을 텍스트로 변환하는 ASR 모델을 지정합니다. |
| 기본값 | `ghost613/faster-whisper-large-v3-turbo-korean` — 한국어 특화 faster-whisper 모델 (GPU: float16, CPU: float32) |
| 사용 위치 | `backend/app/services/asr_service.py` — faster-whisper (CTranslate2), `condition_on_previous_text=False`, `vad_filter=True` |
| 변경 시 | faster-whisper 호환 모델(CTranslate2 형식)로 교체 가능 |

```env
# 예: faster-whisper 공식 large-v3-turbo로 변경
ASR_MODEL=Systran/faster-whisper-large-v3-turbo
```

---

### NMT_ASR_MODEL

| 항목 | 내용 |
|------|------|
| 용도 | ASR→NMT→TTS 실시간 번역 파이프라인 전용 |
| 기본값 | `Helsinki-NLP/opus-mt-ko-en` — 경량 한→영 MarianMT 모델 |
| 속도 요구 | 실시간 (CPU에서 0.1~0.2s) |
| 사용 위치 | `backend/app/services/nmt_service.py` — transformers AutoModelForSeq2SeqLM, `num_beams=1`, `float32` |
| 모델 구조 | NLLB 계열 및 Helsinki(MarianMT) 계열 모두 지원 (`_is_nllb` 플래그로 자동 분기) |

> 슬라이드 번역은 VLM(Qwen3-VL + LoRA)이 담당합니다. NMT 모델은 실시간 음성 번역 전용입니다.

**NMT_ASR 모델 선택 기준**

| 모델 | 실행 위치 | 특징 |
|------|----------|------|
| `Helsinki-NLP/opus-mt-ko-en` | CPU | 경량(~300MB), CPU에서 0.1~0.2s 번역 완료 (기본값) |
| `facebook/nllb-200-distilled-1.3B` | GPU 권장 | 고품질(~2.6GB), CPU에서는 수 초 소요 |

---

### TTS_MODEL

| 항목 | 내용 |
|------|------|
| 사용 이유 | 번역된 영어 텍스트를 음성으로 합성하는 TTS 모델을 지정합니다. |
| 기본값 | `piper` — Piper TTS (en_US-lessac-medium.onnx), 로컬 ONNX 모델 자동 다운로드 |
| 사용 위치 | `backend/app/services/tts_service.py` — piper-tts 패키지 |
| 변경 시 | 현재 Piper 고정 (다른 모델로 변경 시 서비스 코드 수정 필요) |
| 실패 시 | 모델 로드 실패해도 서버는 정상 기동 — TTS 없이 자막만 전송됩니다. |

---

### OCR_MODEL

| 항목 | 내용 |
|------|------|
| 사용 이유 | 문자인식 모델을 지정합니다. 시작 시 로드되어 서비스에 등록됩니다. |
| 기본값 | `rapidocr` — 경량 ONNX 기반, 한국어/영어 지원, CPU 최적화 |
| 사용 위치 | `backend/app/services/ocr_service.py` |
| 슬라이드 오버레이 | 슬라이드 텍스트 추출 및 번역은 VLM(Qwen3-VL)이 담당합니다. OCR은 현재 직접 파이프라인에 사용되지 않습니다. |

---

### ASR_DEVICE / NMT_ASR_DEVICE / TTS_DEVICE / OCR_DEVICE

| 항목 | 내용 |
|------|------|
| 사용 이유 | 각 모델을 GPU(cuda)와 CPU(cpu) 중 어디서 실행할지 지정합니다. |
| 사용 위치 | `backend/app/config.py` — 런타임에 CUDA 설치 여부를 체크해 디바이스 결정 |
| 자동 폴백 | `cuda`로 설정해도 CUDA 미설치 환경이면 자동으로 `cpu`로 전환됩니다. |

| 값 | 설명 |
|----|------|
| `cuda` | NVIDIA GPU 사용. CUDA 및 드라이버 설치 필요 |
| `cpu` | CPU 사용. GPU 없는 환경에서 사용 |

> ASR는 GPU 가속이 실시간 처리에 중요하므로 `cuda`를 권장합니다. NMT(Helsinki opus-mt-ko-en)는 경량 모델로 CPU에서도 실시간 처리가 가능하므로 `cpu`로 설정합니다. TTS(Piper)와 OCR(RapidOCR)은 ONNX CPU 전용 모델이므로 반드시 `cpu`로 설정하세요.

---

### BACKEND_URL (선택)

| 항목 | 내용 |
|------|------|
| 사용 이유 | Vite 개발 서버가 백엔드 API를 프록시할 주소를 지정합니다. |
| 사용 위치 | `frontend/vite.config.ts` — `/api`, `/slides`, `/health`, `/network`, `/ws` 경로를 이 주소로 프록시 |
| 기본값 | 설정 없으면 `http://localhost:8000`으로 동작 |
| 원격 서버 연결 시 | `.env`에 추가 후 서버 IP로 변경 |

```env
# 원격 서버 연결 시만 추가
BACKEND_URL=http://192.168.0.10:8000
```

> **주의**: 이 변수는 Vite 빌드/개발 서버 실행 시에만 읽히는 서버 사이드 변수입니다. `VITE_` 접두사가 없으므로 클라이언트 번들에 노출되지 않습니다.
> Electron 배포판은 항상 `localhost:8000`을 직접 참조하므로 이 설정이 적용되지 않습니다.

---

### VLM_MAX_GPU_MEMORY (선택)

| 항목 | 내용 |
|------|------|
| 사용 이유 | 슬라이드 번역 VLM(Qwen3-VL)이 사용할 GPU 메모리 최대값을 지정합니다. |
| 기본값 | `6GB` |
| 사용 위치 | `translate_slide_v3.py` |
| 초과분 처리 | GPU 한도를 넘는 레이어는 CPU로 오프로드됩니다. |

```env
VLM_MAX_GPU_MEMORY=6GB   # 기본값 (GPU 6GB 이상 환경)
VLM_MAX_GPU_MEMORY=4GB   # GPU 메모리 여유가 적을 때
```

---

### VLM_USE_4BIT (선택)

| 항목 | 내용 |
|------|------|
| 사용 이유 | VLM 모델을 4비트 양자화(BitsAndBytes)로 로드해 GPU 메모리를 절약합니다. |
| 기본값 | `true` |
| 사용 위치 | `translate_slide_v3.py` |

```env
VLM_USE_4BIT=true    # 권장 (GPU 메모리 절약)
VLM_USE_4BIT=false   # 전체 정밀도 (더 많은 VRAM 필요)
```

---

### SKIP_STARTUP_MODELS (선택)

| 항목 | 내용 |
|------|------|
| 사용 이유 | 시작 시 ASR/NMT/TTS/OCR 로딩을 건너뛰고 즉시 슬라이드 번역 전용 모드로 시작합니다. |
| 기본값 | `false` (모든 모델 로드) |
| 사용 위치 | `backend/app/main.py` |
| 사용 시기 | VLM 번역 테스트만 할 때, 또는 실시간 모델을 로드할 메모리가 부족할 때 |

```env
SKIP_STARTUP_MODELS=true
```

## AI 모델 변경 방법

`.env`에서 모델명만 바꾸고 재시작하면 자동으로 새 모델을 다운로드합니다.

```env
# 예: ASR 모델 변경 (faster-whisper 호환 모델)
ASR_MODEL=Systran/faster-whisper-large-v3-turbo

# 예: 실시간 번역 모델 변경 (NLLB로 교체 시 NMT_ASR_DEVICE=cuda 권장)
NMT_ASR_MODEL=facebook/nllb-200-distilled-600M
```

모델 캐시 위치:
- 개발환경: `backend/cache/huggingface`
- 배포판(exe): `%LOCALAPPDATA%\Aunion AI\cache\huggingface`

---

## 배포판(exe)에서 설정 변경

설치 경로 내 `.env` 파일을 직접 수정합니다.

```
<설치경로>\resources\backend\.env
```

기본 설치 경로 예시:
```
C:\Users\USERNAME\AppData\Local\Programs\Aunion AI\resources\backend\.env
```
