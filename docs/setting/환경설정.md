# 환경설정 가이드

프로젝트 루트(`.env`)의 단일 파일로 백엔드·프론트엔드 모든 설정을 관리합니다.

> **왜 .env로 관리하나요?**  
> 모델명·디바이스가 코드에 하드코딩되어 있으면, 모델을 교체할 때마다 코드를 수정하고 재배포해야 합니다.  
> `.env`로 분리하면 파일 한 줄만 바꾸고 재시작하는 것으로 적용이 끝납니다.  
> 팀원마다 GPU/CPU 환경이 다를 때도 각자 `.env`만 수정하면 되므로 코드 충돌이 없습니다.

---

## .env 전체 항목

```env
# HuggingFace 토큰 (공개 모델만 사용하므로 선택사항)
HF_TOKEN=

# OpenAI API 키 (선택사항 — 슬라이드 번역 시 용어집 자동 생성)
OPENAI_API_KEY=

ASR_MODEL=models/whisper-large-v3-turbo-ct2-int8
NMT_ASR_MODEL=Helsinki-NLP/opus-mt-ko-en
OCR_MODEL=surya

ASR_DEVICE=cuda
NMT_ASR_DEVICE=cpu
OCR_DEVICE=cpu

# 서버 시작 시 모델 로딩 여부
# false: 실시간 번역 전체 기능 (ASR + NMT + OCR 로드, 기본값)
# true:  슬라이드 번역 전용 모드
SKIP_STARTUP_MODELS=false

# VLM 설정 (슬라이드 번역)
VLM_BASE_MODEL=models/qwen2.5-vl-7b-instruct
VLM_DEVICE=cuda
VLM_MAX_GPU_MEMORY=4GB
VLM_USE_4BIT=true
```

---

## 항목별 설명

### HF_TOKEN

| 항목 | 내용 |
|------|------|
| 사용 이유 | HuggingFace 모델 다운로드 속도 향상 및 gated 모델 인증에 사용됩니다. |
| 필수 여부 | 기본 모델은 모두 공개 모델이므로 **없어도 동작**합니다. 다운로드 속도가 느리면 발급 권장. |
| 발급 방법 | https://huggingface.co/settings/tokens 에서 발급 |
| 주의 | `.gitignore`에 등록되어 있습니다. 절대 커밋하지 마세요. |

---

### OPENAI_API_KEY

| 항목 | 내용 |
|------|------|
| 사용 이유 | 슬라이드 번역 시 OCR 결과로부터 도메인 용어집(한↔영)을 자동 생성하는 데 사용됩니다. |
| 필수 여부 | **선택사항**. 키가 없으면 용어집 빌드를 건너뛰고 번역을 진행합니다 (번역 품질에 영향 있음). |
| 발급 방법 | https://platform.openai.com/api-keys 에서 발급 |
| 사용 위치 | `backend/app/services/glossary_builder.py` (`translate_slide_v3.py`에서 호출) |
| 주의 | `.gitignore`에 등록되어 있습니다. 절대 커밋하지 마세요. |

---

### ASR_MODEL

| 항목 | 내용 |
|------|------|
| 사용 이유 | 강의자의 한국어 음성을 텍스트로 변환하는 ASR 모델을 지정합니다. |
| 기본값 | `models/whisper-large-v3-turbo-ct2-int8` — 로컬 CT2 변환본 (setup 시 자동 변환) |
| 사용 위치 | `backend/app/services/asr_service.py` — faster-whisper (CTranslate2), `condition_on_previous_text=False`, `vad_filter=True` |
| GPU | cuda: float16 / CPU: float32 자동 선택 |
| 변경 시 | 로컬 경로 또는 HF repo_id 모두 허용. faster-whisper 호환 모델(CTranslate2 형식)로 교체 가능 |

```env
# 예: HuggingFace repo_id로 직접 변환 없이 사용
ASR_MODEL=openai/whisper-large-v3-turbo
```

---

### NMT_ASR_MODEL

| 항목 | 내용 |
|------|------|
| 용도 | ASR → NMT 실시간 번역 파이프라인 전용 |
| 기본값 | `Helsinki-NLP/opus-mt-ko-en` — 경량 한→영 MarianMT 모델 |
| 실행 엔진 | **CTranslate2 CT2 우선** (int8 양자화, CPU 3~5배 가속) → HuggingFace transformers 폴백 |
| CT2 모델 경로 | `models/opus-mt-ko-en-ct2/` (`npm run setup` 시 자동 변환) |
| 속도 | CPU에서 0.05~0.1s |
| 사용 위치 | `backend/app/services/nmt_service.py` |

> 슬라이드 번역은 VLM(Qwen2.5-VL)이 담당합니다. NMT 모델은 실시간 음성 번역 전용입니다.

---

### OCR_MODEL

| 항목 | 내용 |
|------|------|
| 사용 이유 | 문자인식 모델을 지정합니다. 시작 시 로드되어 서비스에 등록됩니다. |
| 기본값 | `surya` — Surya OCR Transformer 기반, GPU 최적화 |
| 모델 | HuggingFace 캐시 자동 다운로드 (`backend/cache/huggingface`) |
| 사용 위치 | `backend/app/services/ocr_service.py` |

---

### ASR_DEVICE / NMT_ASR_DEVICE / OCR_DEVICE

| 항목 | 내용 |
|------|------|
| 사용 이유 | 각 모델을 GPU(cuda)와 CPU(cpu) 중 어디서 실행할지 지정합니다. |
| 사용 위치 | `backend/app/config.py` — 런타임에 CUDA 설치 여부를 체크해 디바이스 결정 |
| 자동 폴백 | `cuda`로 설정해도 CUDA 미설치 환경이면 자동으로 `cpu`로 전환됩니다. |

| 값 | 설명 |
|----|------|
| `cuda` | NVIDIA GPU 사용. CUDA 및 드라이버 설치 필요 |
| `cpu` | CPU 사용. GPU 없는 환경에서 사용 |

> ASR는 GPU 가속이 실시간 처리에 중요하므로 `cuda`를 권장합니다.  
> NMT(Helsinki opus-mt → CT2)는 CPU에서도 충분히 빠르므로 `cpu`로 설정합니다.  
> OCR(Surya)는 GPU 가속을 지원하며, `cuda`를 권장합니다.

---

### SKIP_STARTUP_MODELS

| 항목 | 내용 |
|------|------|
| 사용 이유 | 시작 시 ASR/NMT/OCR 로딩을 건너뛰어 슬라이드 번역 전용 모드로 시작합니다. |
| 기본값 | `false` (모든 모델 로드) |
| 사용 위치 | `backend/app/main.py` |
| 사용 시기 | VLM 번역 테스트만 할 때, 또는 실시간 모델을 로드할 메모리가 부족할 때 |

```env
SKIP_STARTUP_MODELS=true
```

---

### VLM 관련 설정

| 항목 | 기본값 | 설명 |
|------|--------|------|
| `VLM_BASE_MODEL` | `models/qwen2.5-vl-7b-instruct` | VLM 베이스 모델 (로컬 경로 또는 HF repo_id) |
| `VLM_DEVICE` | `cuda` | VLM 실행 디바이스 (CUDA 필수) |
| `VLM_MAX_GPU_MEMORY` | `4GB` | GPU 메모리 한도 (초과분 CPU 오프로드) |
| `VLM_USE_4BIT` | `true` | 4bit 양자화 (BitsAndBytes, GPU 메모리 절약) |

```env
VLM_MAX_GPU_MEMORY=4GB   # GPU 메모리 여유가 적을 때
VLM_USE_4BIT=false       # 전체 정밀도 (더 많은 VRAM 필요)
```

---

### BACKEND_URL (선택)

| 항목 | 내용 |
|------|------|
| 사용 이유 | Vite 개발 서버가 백엔드 API를 프록시할 주소를 지정합니다. |
| 사용 위치 | `frontend/vite.config.ts` — `/api`, `/slides`, `/health`, `/network`, `/transcripts`, `/ws` 경로를 이 주소로 프록시 |
| 기본값 | 설정 없으면 `http://localhost:8000`으로 동작 |

```env
# 원격 서버 연결 시만 추가
BACKEND_URL=http://192.168.0.10:8000
```

> **주의**: 이 변수는 Vite 빌드/개발 서버 실행 시에만 읽히는 서버 사이드 변수입니다.  
> `VITE_` 접두사가 없으므로 클라이언트 번들에 노출되지 않습니다.  
> Electron 배포판은 항상 `127.0.0.1:8000`을 직접 참조하므로 이 설정이 적용되지 않습니다.

## 모델 캐시 위치

| 항목 | 개발 환경 | 배포판(exe) |
|------|-----------|-------------|
| HuggingFace 모델 | `backend/cache/huggingface` | `%LOCALAPPDATA%\Aunion AI\cache\huggingface` |
| NMT CT2 변환본 | `models/opus-mt-ko-en-ct2/` | 동일 |

> **TTS 모델**: 백엔드 TTS가 제거되어 별도 모델 파일이 불필요합니다.  
> 프론트엔드 WASM TTS(piper-tts-web)가 사용하는 모델 파일은 `frontend/public/piper/`, `frontend/public/onnx/`, `frontend/public/worker/`에 위치합니다 (gitignore 대상, 별도 획득 필요).

---

## 배포판(exe)에서 설정 변경

설치 경로 내 `.env` 파일을 직접 수정합니다.

```
<설치경로>\resources\backend\.env
```

기본 설치 경로 예시:
```
C:\Users\USERNAME\AppData\Local\Programs\Aunion AI\resources\backend\.env
```
