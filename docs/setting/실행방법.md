# 실행 방법

> 모든 명령어는 **프로젝트 루트**에서 실행합니다.

---

## 사전 준비

### 필수 도구

| 도구 | 버전 | 설치 확인 |
|------|------|-----------|
| Node.js | 18+ | `node --version` |
| conda | Miniconda / Anaconda | `conda --version` |
| NVIDIA 드라이버 | CUDA 12.6 호환 | `nvidia-smi` |

## 초기 설치 (최초 1회)

```bash
npm run setup
```

이 명령어 하나로 아래 모든 과정이 자동으로 완료됩니다:

| 단계 | 내용 |
|------|------|
| 1 | 루트 npm 패키지 설치 (concurrently 등) |
| 2 | 프론트엔드 npm 패키지 설치 + VAD ONNX 모델 복사 |
| 3 | `.env` 파일 자동 생성 (`.env.example` 복사) |
| 4 | conda `aunion` 환경 생성 (Python 3.11) |
| 5 | Python 패키지 설치: `requirements.txt` → numpy/Pillow 버전 고정 → CUDA 12.6 PyTorch |
| 6 | AI 모델 다운로드 (HuggingFace + CTranslate2 변환) |
| 7 | 설치 검증 (필수 파일 존재 확인) |

> **설치 시간**: 최초 1회 약 20~40분 소요 (AI 모델 다운로드 포함, 약 17.5GB)

---

## 개발 서버 실행

```bash
npm run dev
```

- 프론트엔드: `http://localhost:3000`
- 백엔드: `http://localhost:8000`

백엔드(`conda run -n aunion python backend/run.py`)와 프론트엔드(`vite`)가 동시에 실행됩니다.  
프론트엔드는 백엔드 `/health` 응답을 기다린 후 자동으로 시작됩니다.

---

## .env 설정

`npm run setup` 실행 시 `.env.example`이 `.env`로 자동 복사됩니다.  
기본값으로 바로 사용 가능하며, 필요 시 아래 항목만 수정하세요:

```env
# GPU 없는 환경이면 cpu로 변경
ASR_DEVICE=cuda
NMT_ASR_DEVICE=cpu
OCR_DEVICE=cpu

# HuggingFace 다운로드 속도 향상 (선택)
HF_TOKEN=
```

자세한 항목 설명은 [환경설정](환경설정.md)을 참고하세요.

## Electron 배포 빌드

```bash
# 1. 백엔드 exe 빌드
conda activate aunion
cd backend && pyinstaller aunion.spec && cd ..

# 2. Electron 패키징
npm run electron:build
```

결과물: `setup/` 폴더에 NSIS 인스톨러(`.exe`) 생성

## 문제 해결

### Windows 심볼릭 링크 오류 (WinError 1314)

설정 → 시스템 → 개발자용 → **개발자 모드 ON** 후 `backend/cache/huggingface` 폴더 삭제 후 재실행

### 로딩 화면에서 멈춤

최초 실행 시 모델 다운로드 중일 수 있습니다. 로딩 화면 하단 로그에서 진행 상황 확인 가능.

| 모델 | 용량 | 비고 |
|------|------|------|
| ASR (openai/whisper-large-v3-turbo → CT2 int8) | ~800MB | `models/whisper-large-v3-turbo-ct2-int8/` |
| NMT (facebook/nllb-200-distilled-600M → CT2 int8) | ~600MB | `models/nllb-200-distilled-600M-ct2/` |
| OCR (Surya OCR) | ~500MB | HuggingFace 캐시 (`backend/cache/huggingface`) |
| VLM Base (Qwen2.5-VL-7B) | ~14GB | `models/qwen2.5-vl-7b-instruct/` |
| WASM TTS (piper-tts-web) | ~110MB | `frontend/public/onnx/`, `frontend/public/piper/`, `frontend/public/worker/` (`npm run setup` 시 자동 복사) |

### CUDA 오류

`.env`에서 해당 모델 DEVICE를 `cpu`로 변경 후 재실행:

```env
ASR_DEVICE=cpu
NMT_ASR_DEVICE=cpu
OCR_DEVICE=cpu
```

### ct2-transformers-converter 오류

NMT CT2 변환 실패 시:

```bash
conda run -n aunion pip install ctranslate2
conda run -n aunion ct2-transformers-converter \
  --model facebook/nllb-200-distilled-600M \
  --output_dir models/nllb-200-distilled-600M-ct2 \
  --copy_files tokenizer.json sentencepiece.bpe.model special_tokens_map.json tokenizer_config.json \
  --quantization int8
```

### WASM TTS 미동작 (수강자 화면에서 소리 없음)

`frontend/public/onnx/`, `frontend/public/piper/`, `frontend/public/worker/` 파일이 없으면
piper-tts-web 초기화가 실패합니다. `npm run setup` 을 다시 실행하세요 — 자동으로
`node_modules/piper-tts-web/dist/` 에서 복사됩니다.

### 에러 로그 위치

개발 환경:
```
backend/error_log.txt
```

배포판(exe):
```
%LOCALAPPDATA%\Aunion AI\error_log.txt
```
