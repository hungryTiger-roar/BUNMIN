# 실행 방법

> 모든 명령어는 **프로젝트 루트**에서 실행합니다.

---

## 사전 준비

### 1. conda 환경 생성 (최초 1회)

```bash
conda create -n aunion python=3.10
conda activate aunion
```

> **faster-whisper 주의**: ASR 모델(`ghost613/faster-whisper-large-v3-turbo-korean`)은 `faster-whisper` 패키지가 필요합니다.
> 아래 `npm run setup` 으로 설치하면 `requirements.txt`에 명시된 버전이 자동으로 설치됩니다.

### 2. 의존성 설치 (최초 1회)

루트에서 아래 명령어로 frontend npm, CUDA torch, backend pip를 한 번에 설치합니다:

```bash
npm install          # 루트 의존성 (concurrently 등 dev 도구)
npm run setup        # frontend npm + aunion 환경에 CUDA torch(cu124) + backend requirements.txt
```

> **`npm run setup`이 하는 일**:
> 1. `frontend/` npm install
> 2. aunion 환경에서 기존 torch 제거 후 CUDA 12.4용 torch 재설치
> 3. `backend/requirements.txt` pip install

**GPU 없는 환경 (CPU 전용)**

`npm run setup`은 CUDA torch를 설치합니다. CPU 전용 환경이라면 수동으로 설치하세요:

```bash
npm install
npm install --prefix frontend
conda run --no-capture-output -n aunion pip install -r backend/requirements.txt
```

CUDA 설치 확인:
```bash
conda run -n aunion python -c "import torch; print(torch.cuda.is_available())"  # True 확인
```

### 3. 루트 `.env` 파일 생성

프로젝트 루트(`C:\...\S14P31S205\.env`)에 파일을 생성합니다.
자세한 항목 설명은 [환경설정](환경설정.md)을 참고하세요.

```env
ASR_MODEL=ghost613/faster-whisper-large-v3-turbo-korean
NMT_ASR_MODEL=Helsinki-NLP/opus-mt-ko-en
TTS_MODEL=piper
OCR_MODEL=rapidocr

ASR_DEVICE=cuda
NMT_ASR_DEVICE=cpu
TTS_DEVICE=cpu
OCR_DEVICE=cpu

# HuggingFace gated 모델 사용 시에만 필요
# HF_TOKEN=hf_xxxxxxxxxxxx
```

## 웹 개발 환경 (팀원 전체)

사전 준비 완료 후 루트에서 명령어 하나로 프론트엔드와 백엔드를 동시에 실행합니다.

```bash
npm run dev
```

- 프론트엔드: `http://localhost:3000`
- 백엔드: `http://localhost:8000`

> `npm run dev`는 내부적으로 `conda run -n aunion python backend/run.py`를 실행합니다. Python 경로를 별도로 수정할 필요가 없습니다.

---

## Electron 개발 환경

```bash
npm run electron:dev
```

- Vite dev 서버 실행 후 Electron 창이 자동으로 열립니다.
- `NODE_ENV=development`가 자동으로 설정됩니다.

## Electron 배포 빌드

```bash
# 1. 백엔드 exe 빌드
conda activate aunion
cd backend && pyinstaller aunion.spec && cd ..

# 2. Electron 패키징
npm run electron:build
```

결과물: `setup/` 폴더에 NSIS 인스톨러(`.exe`) 생성

## AI 모델 평가 환경 준비

`npm run dev` 실행 후 평가까지 하려면 추가로 아래를 준비하세요.  
평가에 필요한 패키지(`pymupdf`, `nltk`, `bert-score` 등)는 `requirements.txt`에 포함되어 있어 별도 설치가 필요하지 않습니다.

### 1. 평가 데이터셋 준비

`evaluation/datasets/`는 .gitignore에 포함되어 있습니다.  
팀 공유 스토리지(Google Drive 등)에서 `evaluation/datasets.zip`(약 600MB)을 받아 압축 해제합니다.

```bash
# evaluation/ 폴더에 datasets.zip을 넣은 뒤 실행
cd evaluation
python -c "import zipfile; zipfile.ZipFile('datasets.zip').extractall('datasets')"
cd ..
```

### 2. OCR 데이터셋 이미지 생성

PDF → PNG 변환이 필요합니다 (최초 1회, 약 3~5분 소요).

```bash
python evaluation/rebuild_ocr_dataset.py
```

### 3. 평가 실행

```bash
python -m evaluation.run_eval
# 인터랙티브 메뉴에서 CPU/GPU 및 모델 선택
```

자세한 내용은 [AI 모델 평가 시스템](../evaluation/평가시스템.md)을 참고하세요.

## 문제 해결

**Windows 심볼릭 링크 오류 (WinError 1314)**
설정 → 시스템 → 개발자용 → 개발자 모드 ON 후 `backend/cache/huggingface` 폴더 삭제 후 재실행

**로딩 화면에서 멈춤**
최초 실행 시 모델 다운로드 중일 수 있습니다. 로딩 화면 하단 로그에서 진행 상황 확인 가능

| 모델 | 용량 | 비고 |
|------|------|------|
| ASR (ghost613/faster-whisper-large-v3-turbo-korean) | ~3.1 GB | HuggingFace 자동 다운로드 |
| NMT-ASR (Helsinki-NLP/opus-mt-ko-en) | ~300 MB | HuggingFace 자동 다운로드 |
| TTS (Piper) | ~60 MB | 서비스 초기화 시 자동 다운로드 |
| OCR (Korean PP-OCRv4) | ~수 MB | 패키지에 포함 |

**ASR ImportError**
`faster-whisper` 패키지가 없을 때 발생합니다:
```bash
conda run -n aunion pip install faster-whisper
```

**CUDA 오류**
`.env`에서 해당 모델 DEVICE를 `cpu`로 변경 후 재실행
```env
ASR_DEVICE=cpu
NMT_ASR_DEVICE=cpu
```

**에러 로그 위치**
```
%LOCALAPPDATA%\Aunion AI\error_log.txt
```
