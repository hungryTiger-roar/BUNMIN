## 초기 설치 (최초 1회)

```bash
npm run setup
```

이 명령어 하나로 아래 모든 과정이 자동으로 완료됩니다:

| 단계 | 내용 |
|------|------|
| 1 | 루트 npm 패키지 설치 (concurrently 등) |
| 2 | 프론트엔드 npm 패키지 설치 + VAD ONNX 모델 복사 |
| 3 | `.env` 파일 자동 생성 (`.env.example` 복사) |
| 4 | conda `aunion` 환경 생성 (Python 3.11) |
| 5 | Python 패키지 설치: `requirements.txt` → numpy/Pillow 버전 고정 → CUDA 12.6 PyTorch |
| 6 | AI 모델 다운로드 (HuggingFace + CTranslate2 변환) |
| 7 | 설치 검증 (필수 파일 존재 확인) |

> **설치 시간**: 최초 1회 약 20~40분 소요 (AI 모델 다운로드 포함, 약 20GB)

---

## piper-tts-web WASM 파일 설치 (최초 1회)

수강자 브라우저 WASM TTS가 사용하는 런타임 파일은 용량이 크므로(~110MB) git에서 제외됩니다.
`npm run setup` 완료 후 아래 방법으로 직접 복사합니다.

`node_modules/piper-tts-web/dist/` 내의 `onnx/`, `piper/`, `worker/` 디렉터리를
`frontend/public/` 하위로 복사합니다.

| 대상 경로 | 내용 | 크기 |
|-----------|------|------|
| `frontend/public/onnx/` | ONNX Runtime WASM 런타임 | ~32MB |
| `frontend/public/piper/` | Piper 음소화 엔진 WASM + 데이터 | ~18MB |
| `frontend/public/worker/` | Piper 내부 Web Worker JS | ~60MB |

## 개발 서버 실행

```bash
npm run dev
```

- 프론트엔드: `http://localhost:3000`
- 백엔드: `http://localhost:8000`

백엔드(`conda run -n aunion python backend/run.py`)와 프론트엔드(`vite`)가 동시에 실행됩니다.  
프론트엔드는 백엔드 `/health` 응답을 기다린 후 자동으로 시작됩니다.

## .env 설정

`npm run setup` 실행 시 `.env.example`이 `.env`로 자동 복사됩니다.  
기본값으로 바로 사용 가능하며, 필요 시 아래 항목만 수정하세요:

```env
# GPU 없는 환경이면 cpu로 변경
ASR_DEVICE=cuda
NMT_ASR_DEVICE=cpu
OCR_DEVICE=cpu

# HuggingFace 다운로드 속도 향상 (선택)
HF_TOKEN=
```

자세한 항목 설명은 [환경설정](환경설정.md)을 참고하세요.

## AI 모델 평가 환경

`npm run dev` 실행 후 평가까지 하려면 추가로 아래를 준비하세요.

### 1. 평가 데이터셋 준비

`evaluation/datasets/`는 `.gitignore`에 포함되어 있습니다.  
팀 공유 스토리지(Google Drive 등)에서 `evaluation/datasets.zip`(약 600MB)을 받아 압축 해제합니다.

```bash
cd evaluation
python -c "import zipfile; zipfile.ZipFile('datasets.zip').extractall('datasets')"
cd ..
```

### 2. OCR 데이터셋 이미지 생성

```bash
python evaluation/rebuild_ocr_dataset.py
```

### 3. 평가 실행

```bash
python -m evaluation.run_eval
```

자세한 내용은 [AI 모델 평가 시스템](../evaluation/평가시스템.md)을 참고하세요.

## 문제 해결

### Windows 심볼릭 링크 오류 (WinError 1314)

설정 → 시스템 → 개발자용 → **개발자 모드 ON** 후 `backend/cache/huggingface` 폴더 삭제 후 재실행

### 로딩 화면에서 멈춤

최초 실행 시 모델 다운로드 중일 수 있습니다. 로딩 화면 하단 로그에서 진행 상황 확인 가능.

| 모델 | 용량 | 비고 |
|------|------|------|
| ASR (openai/whisper-large-v3-turbo → CT2 int8) | ~800MB | `models/whisper-large-v3-turbo-ct2-int8/` |
| NMT (Helsinki-NLP/opus-mt-ko-en → CT2 int8) | ~150MB | `models/opus-mt-ct2/` |
| OCR (Korean PP-OCRv4) | ~20MB | HuggingFace 캐시 (`backend/cache/huggingface`) |
| VLM Base (Qwen3-VL-8B) | ~16GB | `models/qwen3-vl-8b-instruct/` |
| VLM LoRA | ~665MB | `models/qwen3/` (git LFS) |
| WASM TTS (piper-tts-web) | ~110MB | `frontend/public/onnx/`, `frontend/public/piper/`, `frontend/public/worker/` (gitignore, 별도 설치) |

### CUDA 오류

`.env`에서 해당 모델 DEVICE를 `cpu`로 변경 후 재실행:

```env
ASR_DEVICE=cpu
NMT_ASR_DEVICE=cpu
OCR_DEVICE=cpu
```

### ct2-transformers-converter 오류

NMT CT2 변환 실패 시:

```bash
conda run -n aunion pip install ctranslate2
conda run -n aunion ct2-transformers-converter \
  --model Helsinki-NLP/opus-mt-ko-en \
  --output_dir models/opus-mt-ct2 \
  --quantization int8
```

### WASM TTS 미동작 (수강자 화면에서 소리 없음)

`frontend/public/onnx/`, `frontend/public/piper/`, `frontend/public/worker/` 파일이 없으면  
piper-tts-web 초기화가 실패합니다. [piper-tts-web WASM 파일 설치](#piper-tts-web-wasm-파일-설치-최초-1회) 단계를 재실행하세요.

### 에러 로그 위치

개발 환경:
```
backend/error_log.txt
```

배포판(exe):
```
%LOCALAPPDATA%\Aunion AI\error_log.txt
```
