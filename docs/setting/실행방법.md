# 실행 방법

> 모든 명령어는 **프로젝트 루트**에서 실행합니다.

---

## 사전 준비

### 1. conda 환경 생성 (최초 1회)

```bash
conda create -n aunion python=3.10
conda activate aunion
pip install -r backend/requirements.txt
```

> **transformers 버전 주의**: ASR 모델(`CohereLabs/cohere-transcribe-03-2026`)은 transformers 4.50 이상이 필요합니다.
> `pip install -r` 후 아래 명령으로 확인·업그레이드하세요.
> ```bash
> pip install "transformers>=4.50.0"
> ```

**GPU(CUDA) 사용 시 추가 설치**

`pip install -r`로 설치되는 torch는 CPU 전용입니다. GPU를 사용하려면 CUDA 버전으로 재설치하세요.

```bash
# CUDA 버전에 맞는 명령어 선택 (nvidia-smi 우측 상단 숫자 기준)
# CUDA 11.8     → cu118
# CUDA 12.1     → cu121
# CUDA 12.4 이상 또는 CUDA 13.x → cu124  ← 최신 드라이버는 이것 사용
pip install torch --index-url https://download.pytorch.org/whl/cu124

# 설치 확인
python -c "import torch; print(torch.cuda.is_available())"  # True 확인
```

CUDA 버전 확인: `nvidia-smi` 명령어의 우측 상단 숫자 기준

### 2. Node 의존성 설치 (최초 1회)

```bash
npm install
cd frontend && npm install && cd ..
```

### 3. Python 경로 수정

`package.json`의 `dev` 스크립트에서 Python 경로를 본인의 conda 환경에 맞게 수정합니다.

```json
"dev": "concurrently \"npm run dev --prefix frontend\" \"C:/Users/본인이름/miniforge3/envs/aunion/python.exe backend/run.py\""
```

현재 conda 환경의 Python 경로 확인:

```bash
conda activate aunion
where python
```

### 4. 루트 `.env` 파일 생성

프로젝트 루트(`C:\...\S14P31S205\.env`)에 파일을 생성합니다.
자세한 항목 설명은 [환경설정](환경설정.md)을 참고하세요.

```env
# 필수: CohereLabs ASR 모델은 gated 모델 — HuggingFace에서 접근 승인 후 토큰 입력
HF_TOKEN=hf_xxxxxxxxxxxx

ASR_MODEL=CohereLabs/cohere-transcribe-03-2026
NMT_ASR_MODEL=facebook/nllb-200-distilled-1.3B
NMT_OCR_MODEL=tencent/HY-MT1.5-1.8B
TTS_MODEL=piper
OCR_MODEL=rapidocr

ASR_DEVICE=cuda
NMT_ASR_DEVICE=cuda
NMT_OCR_DEVICE=cuda
TTS_DEVICE=cpu
OCR_DEVICE=cpu
```

## 웹 개발 환경 (팀원 전체)

사전 준비 완료 후 루트에서 명령어 하나로 프론트엔드와 백엔드를 동시에 실행합니다.

```bash
npm run dev
```

- 프론트엔드: `http://localhost:3000`
- 백엔드: `http://localhost:8000`

---

## Electron 개발 환경

```bash
npm run electron:dev
```

- Vite dev 서버 실행 후 Electron 창이 자동으로 열립니다.
- `NODE_ENV=development`가 자동으로 설정됩니다.

## Electron 배포 빌드

```bash
# 1. 백엔드 exe 빌드
conda activate aunion
cd backend && pyinstaller aunion.spec && cd ..

# 2. Electron 패키징
npm run electron:build
```

결과물: `setup/` 폴더에 NSIS 인스톨러(`.exe`) 생성

## AI 모델 평가 환경 준비

`npm run dev` 실행 후 평가까지 하려면 추가로 아래를 준비하세요.  
평가에 필요한 패키지(`pymupdf`, `nltk`, `bert-score` 등)는 `requirements.txt`에 포함되어 있어 별도 설치가 필요하지 않습니다.

### 1. 평가 데이터셋 준비

`evaluation/datasets/`는 .gitignore에 포함되어 있습니다.  
팀 공유 스토리지(Google Drive 등)에서 `evaluation/datasets.zip`(약 600MB)을 받아 압축 해제합니다.

```bash
# evaluation/ 폴더에 datasets.zip을 넣은 뒤 실행
cd evaluation
python -c "import zipfile; zipfile.ZipFile('datasets.zip').extractall('datasets')"
cd ..
```

### 2. OCR 데이터셋 이미지 생성

PDF → PNG 변환이 필요합니다 (최초 1회, 약 3~5분 소요).

```bash
python evaluation/rebuild_ocr_dataset.py
```

### 3. 평가 실행

```bash
python -m evaluation.run_eval
# 인터랙티브 메뉴에서 CPU/GPU 및 모델 선택
```

자세한 내용은 [AI 모델 평가 시스템](../evaluation/평가시스템.md)을 참고하세요.

## 문제 해결

**Windows 심볼릭 링크 오류 (WinError 1314)**
설정 → 시스템 → 개발자용 → 개발자 모드 ON 후 `backend/cache/huggingface` 폴더 삭제 후 재실행

**로딩 화면에서 멈춤**
최초 실행 시 모델 다운로드 중일 수 있습니다. 로딩 화면 하단 로그에서 진행 상황 확인 가능

| 모델 | 용량 | 비고 |
|------|------|------|
| ASR (Cohere) | ~3 GB | HuggingFace 자동 다운로드 |
| NMT-ASR (nllb-200-distilled-1.3B) | ~2.6 GB | HuggingFace 자동 다운로드 |
| NMT-OCR (HY-MT1.5-1.8B) | ~3.6 GB | HuggingFace 자동 다운로드 |
| TTS (Piper) | ~60 MB | 서비스 초기화 시 자동 다운로드 |
| OCR (Korean PP-OCRv4) | ~수 MB | HuggingFace 자동 다운로드 |

**ASR ImportError**
transformers 버전이 낮거나 패키지 문제일 때 발생합니다:
```bash
pip install "transformers>=4.50.0"
```

**CUDA 오류**
`.env`에서 해당 모델 DEVICE를 `cpu`로 변경 후 재실행
```env
ASR_DEVICE=cpu
NMT_ASR_DEVICE=cpu
NMT_OCR_DEVICE=cpu
```

**에러 로그 위치**
```
%LOCALAPPDATA%\Aunion AI\error_log.txt
```
