## 초기 설치 (최초 1회)

```bash
npm run setup
```

이 명령어 하나로 아래 모든 과정이 자동으로 완료됩니다:

| 단계 | 내용 |
|------|------|
| 1 | 루트 npm 패키지 설치 (concurrently 등) |
| 2 | 프론트엔드 npm 패키지 설치 + VAD ONNX 모델 복사 |
| 3 | `.env` 파일 자동 생성 (`.env.example` 복사) |
| 4 | conda `aunion` 환경 생성 (Python 3.11) |
| 5 | Python 패키지 설치: `requirements.txt` → numpy/Pillow 버전 고정 → CUDA 12.6 PyTorch |
| 6 | AI 모델 다운로드 (HuggingFace + CTranslate2 변환) |
| 7 | 설치 검증 (필수 파일 존재 확인) |

> **설치 시간**: 최초 1회 약 20~40분 소요 (AI 모델 다운로드 포함, 약 10GB)

## 개발 서버 실행

```bash
npm run dev
```

- 프론트엔드: `http://localhost:43000`
- 백엔드: `http://localhost:48000`
- TURN 서버: `:47878` (UDP + TCP) — WebRTC 원본 음성 relay 용. 표준 3478 은 Teams/Zoom 충돌 회피로 임의 포트 사용.

백엔드(`conda run -n aunion python backend/run.py`) · 프론트엔드(`vite`) · TURN 서버(`node turn-server/index.js`) 세 프로세스가 concurrently 로 동시에 실행됩니다.  
프론트엔드는 백엔드 `/health` 응답을 기다린 후 자동으로 시작됩니다.

> **수강자 공유 URL**: 강사 화면의 "초대 링크 복사" 버튼은 **base URL** (`http://<lan_ip>:48000/`) 을 만들어줍니다. 수강자가 이 URL 에 접속하면 이름 입력 화면(`Start`) 이 뜨고, 입장 버튼을 누르면 `/student` 로 이동합니다.

## .env 설정

`npm run setup` 실행 시 `.env.example`이 `.env`로 자동 복사됩니다.  
기본값으로 바로 사용 가능하며, 필요 시 아래 항목만 수정하세요:

```env
# GPU 없는 환경이면 cpu로 변경
ASR_DEVICE=cuda
NMT_ASR_DEVICE=cuda
OCR_DEVICE=cpu

# HuggingFace 다운로드 속도 향상 (선택)
HF_TOKEN=
```

자세한 항목 설명은 [환경설정](환경설정.md)을 참고하세요.

## Electron 배포 빌드

```bash
# 한 줄로: 백엔드 PyInstaller + electron-builder + ISCC 인스톨러
npm run build:installer
```

또는 단계별:

```bash
# 1. 백엔드 exe 빌드
npm run build:backend

# 2. Electron 패키징
npm run electron:build

# 3. Inno Setup 인스톨러 생성
node scripts/run-iscc.js installer.iss
```

결과물:
- `backend/dist/aunion_backend.exe` — PyInstaller 백엔드
- `frontend/setup/win-unpacked/` — electron-builder 출력
- `setup/Aunion-AI-Setup-0.1.0.exe` — Inno Setup 최종 인스톨러

> **ISCC 경로**: `scripts/run-iscc.js` 가 PATH · `%LOCALAPPDATA%\Programs\Inno Setup 6\ISCC.exe` · Program Files 순으로 자동 탐색. winget 의 per-user 설치도 자동으로 잡습니다.

## 문제 해결

### Electron 운영판에서 디버깅 — DevTools 단축키

운영판 (.exe 설치본) 에서도 **`Ctrl+Shift+I`** 또는 **`F12`** 누르면 DevTools 가 열립니다. Console 탭에서 마이크 / WebSocket / 렌더링 에러를 확인할 수 있습니다.

| 단축키 | 동작 |
|--------|------|
| `Ctrl+Shift+I` / `F12` | DevTools 토글 |

`frontend/electron/main.cjs` 의 `before-input-event` 리스너가 keypress 를 가로채 처리. 메뉴바는 여전히 숨겨진 상태 (`Menu.setApplicationMenu(null)`).

### Windows 심볼릭 링크 오류 (WinError 1314)

설정 → 시스템 → 개발자용 → **개발자 모드 ON** 후 `backend/cache/huggingface` 폴더 삭제 후 재실행

### 로딩 화면에서 멈춤

최초 실행 시 모델 다운로드 중일 수 있습니다. 로딩 화면 하단 로그에서 진행 상황 확인 가능.

| 모델 | 용량 | 비고 |
|------|------|------|
| ASR (openai/whisper-large-v3-turbo → CT2 int8) | ~800MB | `models/whisper-large-v3-turbo-ct2-int8/` |
| NMT (facebook/nllb-200-distilled-600M → CT2 int8) | ~600MB | `models/nllb-200-distilled-600M-ct2/` |
| OCR (Surya OCR) | ~500MB | HuggingFace 캐시 (`backend/cache/huggingface`) |
| VLM Base (Qwen3-VL-4B) | ~8GB | `models/qwen3-vl-4b-instruct/` |
| WASM TTS (piper-tts-web) | ~110MB | `frontend/public/onnx/`, `frontend/public/piper/`, `frontend/public/worker/` (`npm run setup` 시 자동 복사) |

### CUDA 오류

`.env`에서 해당 모델 DEVICE를 `cpu`로 변경 후 재실행:

```env
ASR_DEVICE=cpu
NMT_ASR_DEVICE=cpu
OCR_DEVICE=cpu
```

### ct2-transformers-converter 오류

NMT CT2 변환 실패 시:

```bash
conda run -n aunion pip install ctranslate2
conda run -n aunion ct2-transformers-converter \
  --model facebook/nllb-200-distilled-600M \
  --output_dir models/nllb-200-distilled-600M-ct2 \
  --copy_files tokenizer.json sentencepiece.bpe.model special_tokens_map.json tokenizer_config.json \
  --quantization int8
```

### WASM TTS 미동작 (수강자 화면에서 소리 없음)

`frontend/public/onnx/`, `frontend/public/piper/`, `frontend/public/worker/` 파일이 없으면
piper-tts-web 초기화가 실패합니다. `npm run setup` 을 다시 실행하세요 — 자동으로
`node_modules/piper-tts-web/dist/` 에서 복사됩니다.

### 에러 로그 위치

개발 환경:
```
backend/error_log.txt
```

배포판(exe):
```
%LOCALAPPDATA%\Aunion AI\error_log.txt
```
