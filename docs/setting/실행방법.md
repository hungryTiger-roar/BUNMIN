# 실행 방법

> 모든 명령어는 **프로젝트 루트**에서 실행합니다.

---

## 사전 준비

### 1. 의존성 설치

```bash
# 루트 (concurrently, Electron 관련 deps)
npm install

# 프론트엔드
cd frontend && npm install && cd ..
```

### 2. 백엔드 conda 환경 설정

conda 환경이 없는 경우 생성합니다.

```bash
conda create -n aunion python=3.10
conda activate aunion
pip install -r backend/requirements.txt
```

### 3. 루트 `package.json` Python 경로 수정

`package.json`의 `dev` 스크립트에서 Python 경로를 본인의 conda 환경에 맞게 수정합니다.

```json
"dev": "concurrently \"npm run dev --prefix frontend\" \"C:/Users/본인이름/miniforge3/envs/aunion/python.exe backend/run.py\""
```

conda 환경 python 경로 확인 방법:

```bash
conda activate aunion
where python
```

### 4. `backend/.env` 파일 생성

자세한 내용은 [환경설정](환경설정.md)을 참고하세요.

```env
ASR_MODEL=ghost613/faster-whisper-large-v3-turbo-korean
NMT_MODEL=Helsinki-NLP/opus-mt-ko-en
TTS_MODEL=facebook/mms-tts-eng
ASR_DEVICE=cpu
NMT_DEVICE=cpu
TTS_DEVICE=cpu
OCR_DEVICE=cpu
```

---

## 웹 개발 환경 (팀원 전체)

사전 준비 완료 후 루트에서 명령어 하나로 프론트엔드와 백엔드를 동시에 실행합니다.

```bash
npm run dev
```

- 프론트엔드: `http://localhost:3000`
- 백엔드: `http://localhost:8000`

---

## Electron 개발 환경

```bash
npm run electron:dev
```

- Vite dev 서버 실행 후 Electron 창이 자동으로 열립니다.
- `NODE_ENV=development`가 자동으로 설정됩니다.

---

## Electron 배포 빌드

```bash
# 1. 백엔드 exe 빌드
conda activate aunion
cd backend && pyinstaller aunion.spec && cd ..

# 2. Electron 패키징
npm run electron:build
```

결과물: `setup/` 폴더에 NSIS 인스톨러(`.exe`) 생성

---

## 명령어 요약

| 명령어 | 설명 |
|--------|------|
| `npm install` | 루트 의존성 설치 (최초 1회) |
| `npm run dev` | 웹 개발 — 프론트엔드 + 백엔드 동시 실행 |
| `npm run electron:dev` | Electron 개발 |
| `npm run electron:build` | Electron 배포 빌드 |

## 문제 해결

**Windows 심볼릭 링크 오류 (WinError 1314)**
설정 → 시스템 → 개발자용 → 개발자 모드 ON 후 `backend/cache/huggingface` 폴더 삭제 후 재실행

**로딩 화면에서 멈춤**
최초 실행 시 모델 다운로드 중일 수 있습니다 (10~20분 소요). 로딩 화면 하단 로그에서 진행 상황 확인 가능

**CUDA 오류**
`backend/.env`에서 해당 모델 DEVICE를 `cpu`로 변경 후 재실행
```env
ASR_DEVICE=cpu
NMT_DEVICE=cpu
```

**에러 로그 위치**
```
%LOCALAPPDATA%\Aunion AI\error_log.txt
```
