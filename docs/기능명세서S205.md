## 1. Electron 데스크톱 앱

### 1.1 앱 실행 및 초기화

| ID | 분류 | 기능명 | 요구사항 |
|----|------|--------|---------|
| E-FR-001 | 앱실행 | 프로그램 시작 | 사용자는 데스크톱 아이콘을 더블클릭하여 프로그램을 실행할 수 있어야 한다. |
| E-FR-002 | 앱실행 | Electron 앱 로드 | 시스템은 **Electron 41.x** 기반으로 앱을 실행해야 한다. |
| E-FR-003 | 앱실행 | Vite 개발 서버 연결 | 시스템은 개발 모드에서 Vite 개발 서버(**127.0.0.1:43000**)에 연결해야 한다. (표준 3000 충돌 회피) |
| E-FR-004 | 앱실행 | React 앱 로드 | 시스템은 React 18.2.x 기반 웹 앱을 Electron 창에 로드해야 한다. |
| E-FR-005 | 앱실행 | 백엔드 자동 기동 | 시스템은 Electron 메인 프로세스에서 aunion_backend.exe를 `spawn(detached=true, unref)` 로 자동 기동해야 한다. |
| E-FR-006 | 앱실행 | TURN 서버 자체 기동 | 시스템은 사내망(SSAFY 등) P2P 차단 환경 우회를 위해 로컬 TURN 서버(node-turn, **47878**, externalIps=LAN IP)를 자체 기동해야 한다. |
| E-FR-007 | 앱실행 | 백엔드 연결 확인 | 시스템은 백엔드 서버(**127.0.0.1:48000** — 표준 8000 충돌 회피)와 GIL 독립 health 서버(**127.0.0.1:18765**) 양쪽으로 응답을 확인해야 한다. |
| E-FR-008 | 앱실행 | 단일 인스턴스 락 | 시스템은 `app.requestSingleInstanceLock()` 으로 중복 실행 시 두 번째 인스턴스를 즉시 종료하고 기존 창을 활성화해야 한다. |
| E-FR-009 | 앱실행 | 부모 PID 워치독 | 백엔드는 `AUNION_PARENT_PID` 환경 변수로 받은 Electron PID를 2초 간격 폴링하고, 부모 종료 감지 시 학생 자막 다운로드용 **5분 grace** 후 자살한다. |
| E-FR-010A | 앱실행 | 백엔드 재시작 시 이전 인스턴스 정리 | `startBackend()` 진입 직후 `taskkill /F /IM aunion_backend.exe /T` 로 grace 도중인 이전 백엔드를 강제 종료해야 한다. |
| E-FR-010B | 앱실행 | UTF-8 강제 출력 | 시스템은 백엔드 spawn 시 `PYTHONUNBUFFERED=1`, `PYTHONUTF8=1`, `PYTHONIOENCODING=utf-8` 을 설정해 stdout 한글 깨짐을 방지해야 한다. |
| E-FR-010C | 앱실행 | HF 토큰 격리 | 시스템은 시스템 환경변수의 만료/잘못된 `HF_TOKEN` / `HUGGINGFACE_HUB_TOKEN` 을 백엔드 spawn 시 제거해 `.env` 의 정상 토큰이 우선 적용되도록 해야 한다. |

### 1.2 창 관리

| ID | 분류 | 기능명 | 요구사항 |
|----|------|--------|---------|
| E-FR-010 | 창관리 | 메인 창 생성 | 시스템은 앱 시작 시 메인 창을 생성해야 한다. |
| E-FR-011 | 창관리 | 창 크기 설정 | 시스템은 메인 창의 기본 크기를 **1280×800** 픽셀로 설정해야 한다. |
| E-FR-012 | 창관리 | 창 최소 크기 | 시스템은 창의 최소 크기를 **900×600** 픽셀로 제한해야 한다. |
| E-FR-013 | 창관리 | 창 크기 조절 | 사용자는 창의 크기를 자유롭게 조절할 수 있어야 한다 (frame:false 라도 가장자리 8px 리사이즈). |
| E-FR-014 | 창관리 | 창 최대화 | 사용자는 창을 최대화할 수 있어야 한다 (자체 타이틀바 버튼). |
| E-FR-015 | 창관리 | 창 최소화 | 사용자는 창을 최소화할 수 있어야 한다 (자체 타이틀바 버튼). |
| E-FR-016 | 창관리 | 전체 화면 | 사용자는 강의자 발표 모드에서 `document.documentElement.requestFullscreen()` 으로 전체 화면 모드로 전환할 수 있어야 한다. |
| E-FR-017 | 창관리 | 창 상태 저장 | 시스템은 resize/move/maximize/unmaximize 이벤트마다 500ms debounce 로 `%LOCALAPPDATA%\Aunion AI\window-state.json` 에 정상(=unmaximize) bounds 와 maximize 플래그를 저장해야 한다. |
| E-FR-018 | 창관리 | 창 상태 복원 | 시스템은 앱 재실행 시 저장된 좌표가 살아있는 디스플레이 안에 있을 때만 적용하고, 아니면 Electron 의 중앙 배치를 사용해야 한다. (보조 모니터 분리 후 화면 밖 좌표 방지) |
| E-FR-019 | 창관리 | 트레이 hide (X 버튼) | X 버튼은 앱을 종료하지 않고 시스템 트레이로 숨겨야 한다. 단, `isQuitting=true` (트레이 메뉴 "종료" / IPC quit-app / OS 셧다운) 일 때만 실제 종료. |
| E-FR-019A | 창관리 | 자체 타이틀바 | 시스템은 `frame: false` 로 OS 기본 프레임을 제거하고, renderer 가 그린 자체 타이틀바(`components/common/TitleBar.tsx`) + IPC 채널(`window-minimize` / `window-toggle-maximize` / `window-close`) 로 윈도우 컨트롤을 제공해야 한다. |
| E-FR-019B | 창관리 | hidden window throttling 비활성화 | 트레이로 hide 한 상태에서도 OCR 업로드 / fetch / setInterval 이 지연되지 않도록 `webPreferences.backgroundThrottling = false` 를 설정해야 한다. |
| E-FR-019C | 창관리 | 권한 자동 허가 | 마이크/카메라(media), clipboard, fullscreen, mediaKeySystem 권한은 자동 allow (외부 사이트 아닌 단독 앱). |

### 1.9 첫 실행 다운로드 마법사 (NEW)

| ID | 분류 | 기능명 | 요구사항 |
|----|------|--------|---------|

## 5. 실시간 음성 처리

> **아키텍처 개요**: 4채널 신호(강사 원본 음성, 자막, 시각 이벤트, 영어 TTS)가 처리 시간이 다르므로,
> 적응형 동기 지연(`currentDelay`)을 적용하여 학생 측에서 같은 박자로 도착하도록 정렬한다.
>
> **개선 핵심**: ① 처리 시간 단축(8초 강제 분할, NMT beam 4→3, TTS 1.2배속), ② 적응형 지연(2~20초, 최근 20번 실측 P90 + jitter 마진).

### 5.1 음성 캡처

| ID | 분류 | 기능명 | 요구사항 |
|----|------|--------|---------|
| A-FR-001 | 음성캡처 | 마이크 권한 요청 | 시스템은 마이크 사용 전 브라우저 권한을 요청해야 한다 (Electron 자동 allow). |
| A-FR-002 | 음성캡처 | 오디오 컨텍스트 생성 | 시스템은 16kHz 샘플레이트로 AudioContext를 생성해야 한다. |

### 5.8 적응형 동기 지연 (Adaptive Sync Delay)

> **목적**: 4채널 중 가장 느린 신호(보통 TTS)에 맞춰 빠른 신호들을 지연시켜 같은 시각에 도착시킴.
> **구현**: `frontend/src/hooks/useDelayBufferPlayer.ts`

| ID | 분류 | 기능명 | 요구사항 |
|----|------|--------|---------|
| A-FR-070 | 적응형지연 | 실측 처리 시간 수집 | 시스템은 매 발화마다 ASR + NMT + TTS + 네트워크 합계 시간을 측정해야 한다. |
| A-FR-071 | 적응형지연 | 슬라이딩 윈도우 | 시스템은 최근 20번의 실측 처리 시간을 슬라이딩 윈도우로 유지해야 한다. |
| A-FR-072 | 적응형지연 | P90 계산 | 시스템은 윈도우 내 처리 시간의 90 백분위수(P90)를 산출해야 한다. |
| A-FR-073 | 적응형지연 | jitter 마진 | 시스템은 처리 시간 변동성(jitter)이 클수록 자동으로 마진을 증가시켜야 한다. |
| A-FR-074 | 적응형지연 | currentDelay 산정 | 시스템은 `currentDelay = P90 + jitter 마진`으로 `currentDelay`를 산정해야 한다. |
| A-FR-075 | 적응형지연 | 하한 강제 | 시스템은 `currentDelay` 하한을 `VITE_SYNC_DELAY_MS`(기본 **2000ms**)로 강제해야 한다. |
| A-FR-076 | 적응형지연 | 상한 강제 | 시스템은 `currentDelay` 상한을 20000ms (20초)로 강제해야 한다. |
| A-FR-077 | 적응형지연 | 즉시 증가 | 시스템은 처리 시간이 갑자기 늘어나면 `currentDelay`를 즉시 증가시켜야 한다. (어긋남 방지 우선) |
| A-FR-078 | 적응형지연 | 점진적 감소 | 시스템은 `currentDelay` 감소를 점진적으로 수행하여 음성이 빨리 감기처럼 들리지 않도록 해야 한다. (조용한 구간에서 회수) |
| A-FR-079 | 적응형지연 | 무발화 수렴 | 시스템은 발화가 없는 동안 큐가 비면 `currentDelay`가 자연히 하한(2초)으로 수렴하도록 해야 한다. |
| A-FR-080 | 적응형지연 | currentDelay 브로드캐스트 | 시스템은 `currentDelay` 값을 모든 학생 클라이언트에 공유하여 4채널이 동일 값으로 지연되도록 해야 한다. |
| A-FR-081 | 적응형지연 | 로깅 | 시스템은 `currentDelay` 변화와 `[LATENCY]` 분포를 백엔드 콘솔에 로깅해야 한다. |
| A-FR-082 | 적응형지연 | 최소 지연 우선 | 강사-학생 wall-clock 딜레이를 줄이는 방향을 최우선으로 한다. 잘림 / drop / desync 가 발생할 때만 예외적으로 지연을 늘린다. |

## 7. 슬라이드 번역 처리

> **아키텍처 개요**: PDF 강의자료를 한국어 → 영어로 번역하는 파이프라인.
> 핵심 설계: **extract → translate → apply 분리**로 PDF Layer와 OCR 블록을 공통 번역하여 문서 전체 맥락 및 용어 일관성을 확보.
>
> **파이프라인 흐름**: `[Stage 1] 페이지 분류 → [Stage 2] Extract → [Stage 3] Translate → [Stage 4] Apply → [Stage 5] PDF 합성`
