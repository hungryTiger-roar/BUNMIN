# AI 모델 평가 시스템

ASR, NMT, TTS, OCR 모델의 품질(WER, BLEU, CER)과 속도(RTF, 지연시간)를 측정합니다.

---

## 사전 준비

conda 환경이 설치되어 있어야 합니다.

```bash
conda activate aunion
```

> **주의**: 반드시 conda aunion 환경에서 실행해야 합니다.
> `python -m evaluation.run_eval`은 잘못된 Python 환경 감지 시 자동으로 conda aunion으로 전환합니다.

---

## 데이터셋 준비 (최초 1회)

`evaluation/datasets/`는 .gitignore에 포함되어 있어 별도로 받아야 합니다.

### 방법 1. datasets.zip 압축 해제 (권장)

팀 공유 스토리지에서 `evaluation/datasets.zip`(약 600MB)을 받아 루트에서 압축 해제합니다.

```bash
# Windows 탐색기에서 압축 해제하거나 아래 명령 사용
cd <프로젝트루트>/evaluation
# datasets.zip을 evaluation/ 폴더에 넣은 뒤
python -c "import zipfile; zipfile.ZipFile('datasets.zip').extractall('datasets')"
```

해제 후 `evaluation/datasets/` 아래 `asr_samples/`, `nmt_samples/`, `tts_samples/`, `ocr_samples/`가 생성됩니다.

> **OCR 추가 작업 필요**: `ocr_samples/`에는 PDF만 포함됩니다.  
> 아래 [OCR 강의 슬라이드 데이터셋](#ocr-강의-슬라이드-데이터셋) 섹션의 2단계(이미지 변환)를 실행해야 평가가 가능합니다.

### 방법 2. TTS 텍스트 재생성

datasets.zip 압축 해제 후 TTS 텍스트(`tts_samples/texts.json`)만 다시 만들고 싶을 때:

```bash
# nmt_samples/ko_en_pairs.json 이 있어야 실행 가능
python evaluation/download_datasets.py
```

## 평가 실행

프로젝트 루트에서 실행합니다.

```bash
python -m evaluation.run_eval
```

실행 시 인터랙티브 메뉴가 표시됩니다:

```
실행 환경 선택:
  1. CPU
  2. GPU

평가할 모델을 선택하세요:
  1. ASR      (음성인식)
  2. NMT      (번역)
  3. TTS      (음성합성)
  4. OCR      (문자인식)
  5. Pipeline (ASR→NMT→TTS 전체, 속도 측정 + 플로우 확인)
  6. OCR+NMT  (슬라이드 번역)
  7. 전체 평가
  8. 이전 결과와 비교
```

TTS 평가 시 생성된 오디오 파일이 자동 저장됩니다:
`evaluation/datasets/tts_samples/output/sample_NN.wav`

## 특정 모델만 평가

```bash
python -m evaluation.run_eval --model asr
python -m evaluation.run_eval --model nmt
python -m evaluation.run_eval --model tts
python -m evaluation.run_eval --model ocr
```

### NMT XCOMET-XL 평가

기본 NMT 평가는 BLEU, METEOR, BERTScore만 측정합니다.  
XCOMET-XL(사람 평가와 상관관계 가장 높은 지표)을 포함하려면 `--xcomet` 플래그를 추가하세요.

```bash
python -m evaluation.run_eval --model nmt --xcomet
```

> **주의**: XCOMET-XL 평가는 모델 크기(~7GB)로 인해 처음 실행 시 다운로드가 필요하며,  
> 200문장 기준 약 50분 소요됩니다. 모델 접근 승인도 필요합니다.  
> 승인 신청: https://huggingface.co/Unbabel/XCOMET-XL

**사전 설치:**
```bash
pip install unbabel-comet
```

- 최신 결과: `evaluation/results/latest.json`

## 평가 지표

| 모델 | 지표 | 설명 |
|------|------|------|
| ASR | WER | 단어 오류율 (낮을수록 좋음) |
| NMT | BLEU | 번역 품질 — 단어 n-gram 일치율 (동의어 미반영, 낮게 나오는 경향) |
| NMT | METEOR | 번역 품질 — 동의어·어간 고려한 일치율 |
| NMT | BERTScore F1 | 번역 품질 — 의미 기반 유사도 (동의어에 강함) |
| NMT | XCOMET-XL | 번역 품질 — 소스 문장까지 참조, 사람 평가와 상관관계 가장 높음 (`--xcomet` 플래그 필요) |
| TTS | RTF | 실시간 처리 비율 (1.0 이하 = 실시간 가능) |
| OCR | CER | 문자 오류율 (낮을수록 좋음) |
| 전체 | RTF | 실시간 처리 비율 (1.0 이하 = 실시간 가능) |

## OCR 강의 슬라이드 데이터셋

RapidOCR의 한국어 대학교 강의 슬라이드 도메인 성능을 평가하기 위한 별도 데이터셋입니다.

### 데이터셋 구성

[KOCW](https://www.kocw.net) 공개 강의 PDF를 자동 다운로드하여 구성합니다.

| 선별 기준 | 값 |
|-----------|-----|
| 슬라이드당 글자 수 | 50 ~ 500자 |
| 한글 비율 | 40% 이상 |
| 과목당 최대 슬라이드 수 | 5장 |

### 실행 순서

```bash
# 프로젝트 루트에서 실행

# 1. KOCW 한국어 강의 PDF 자동 다운로드 (이미 PDF가 있으면 건너뜀)
#    다운로드 위치: evaluation/datasets/ocr_samples/
python evaluation/download_kocw_pdfs.py

# 2. Ground Truth + 슬라이드 이미지 생성 (pymupdf는 requirements.txt에 포함)
#    ocr_samples/ 의 PDF를 읽어 PNG + ground_truth.json 생성 (PDF는 보존)
python evaluation/rebuild_ocr_dataset.py

# 3. OCR 평가 실행
python -m evaluation.run_eval --model ocr
```

생성 위치: `evaluation/datasets/ocr_samples/` — PDF + PNG 이미지 + `ground_truth.json`
