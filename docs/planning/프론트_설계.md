# 프론트엔드 설계

## 개요

React + TypeScript 기반 웹앱.
Electron 데스크탑 앱으로도 패키징 가능하며, 강의자와 수강자 모두 동일한 앱을 사용하고 역할만 다릅니다.

---

## 기술 스택

| 분류 | 기술 |
|------|------|
| Framework | React + TypeScript |
| 번들러 | Vite |
| 데스크탑 (선택) | Electron (루트 package.json에서 관리) |
| 상태 관리 | Zustand |
| 스타일링 | Tailwind CSS |
| 실시간 통신 | WebSocket (native) |
| WASM TTS | piper-tts-web (CPU ONNX, 메인 스레드) |

## 화면 구성

### 1. 홈 (역할 선택)
- 강의자로 시작 / 수강자로 참여 선택

### 2. 참여 (수강자 이름 입력) — `/#/student/start`
- 이름 입력 후 강의 참여
- 이름 로컬스토리지 저장 옵션
- 한국어 / English UI 전환
- **"강의 참여" 버튼 클릭 시 navigate state(`autoEnter: true`)를 Student 화면에 전달 → 클릭의 transient activation을 활용해 AudioContext 즉시 언락**

### 3. 로딩
- 백엔드 AI 모델 로딩 진행률 표시
- 모델별 로딩 상태 (ASR / NMT-ASR / OCR / VLM)

### 4. 강의자 화면
- **강의자료 모드**: PDF 업로드 → 슬라이드 뷰어로 페이지 제어
- **화면공유 모드**: 화면 캡처 후 수강자에게 실시간 전송
- 마이크 ON/OFF → 음성 실시간 번역
- 강의 시작 / 일시정지 / 종료 제어
- 실시간 자막 미리보기
- 마우스 커서 위치 수강자에게 실시간 공유 (상대 좌표 0~1)

### 5. 수강자 화면
- 슬라이드 또는 화면공유 수신 표시
- **WASM TTS 오디오 재생**: 번역 텍스트를 브라우저 내 piper-tts-web으로 합성
  - Start 페이지 경유 시 즉시 언락 / 직접 접속 시 첫 인터랙션에서 자동 언락
  - 언어 선택: 영어 · 독일어 · 스페인어 · 러시아어 (한국어/off 선택 시 TTS 스킵)
  - 볼륨 슬라이더 · 음소거 제어
- **자막 표시**: 메인 자막 + 부자막 동시 표시, 언어 독립 선택
- 자막 커스터마이징: 글자 크기 · 스타일(기본/테두리/글로우) · 위치(상단/하단)
- 화면 비율 선택: 16:9 / 4:3 / 5:3
- 라이트 / 다크 / 그라데이션 테마 3-모드 토글
- 채팅 (강의자 ↔ 수강자)
- 참여자 목록 (강의자 · 수강자)
- 강의 자료 다운로드 (원본 PDF / 번역 PDF)
- 강의 종료 후 자막 다운로드 모달 (TXT / SRT)
- 강의자 커서 스팟라이트 오버레이 (ref 기반 DOM 직접 조작)

## Electron ↔ 백엔드 통신

```
Electron main.cjs
  ├── spawn aunion_backend.exe
  ├── stdout 파싱 → __AUNION_STATUS__ JSON으로 로딩 진행률 수신
  ├── health 폴링 (backup, port 18765)
  └── IPC: get-lan-ip → LAN IP 반환
```

## WASM TTS (piper-tts-web)

수강자 브라우저 내에서 CPU ONNX로 음성을 합성합니다. 서버에서 음성 파일을 전송하지 않습니다.

### 지원 언어 및 음성 모델

| audioLang | Piper Voice ID | 비고 |
|-----------|---------------|------|
| `en` | en_US-lessac-medium | 영어 (기본) |
| `de` | de_DE-thorsten-medium | 독일어 |
| `es` | es_MX-ald-medium | 스페인어 |
| `ru` | ru_RU-irina-medium | 러시아어 |
| `ko` / `both` / `off` | — | Piper 미지원 → TTS 스킵, 자막만 표시 |

### 아키텍처

```
useWebSocket (onTranslation 콜백)
    │
    │ 번역 텍스트 수신
    ▼
Student.tsx (onTranslation handler)
    │
    │ isAudioUnlocked 확인
    ▼
useTTS.synthesize(text, audioLang)
    │
    ├── synthesizeQueue에 push
    ▼
PiperWebEngine.generate()   ← 메인 스레드 (내부 Worker spawn)
    │
    ▼
AudioContext.decodeAudioData()
    │
    ▼
GainNode → AudioContext.destination
```

### 메모리 관리 (OOM 방지)

- `audioLang` 변경 시 `useEffect` cleanup에서 `engineRef.current = null`
- 이전 엔진의 내부 OnnxWebWorker / PhonemizeWebWorker가 GC 대상이 되어 WASM 힙 해제
- 새 언어 모델 로드 전 이전 모델 메모리가 반환되어 OOM 방지

### 오디오 언락 정책

브라우저 autoplay 정책으로 인해 `AudioContext`는 사용자 제스처 후에만 활성화됩니다.

| 접근 경로 | 언락 시점 |
|-----------|----------|
| Start 페이지 "강의 참여" 클릭 경유 | Student.tsx 진입 직후 즉시 언락 (transient activation 활용) |
| 직접 URL 접속 / 새로고침 | 화면의 첫 클릭 · 키 입력 · 터치 시 자동 언락 (`{ once: true }` 리스너) |

### WASM 런타임 파일

piper-tts-web이 런타임에 fetch하는 파일들은 `public/` 하위에 정적 배치해야 합니다.
크기가 크므로 git에서 제외(`gitignore`)하며, 별도 스크립트로 획득합니다.

| 디렉터리 | 내용 | 크기 |
|----------|------|------|
| `public/onnx/` | ONNX Runtime WASM | ~32MB |
| `public/piper/` | Piper 음소화 엔진 WASM + 데이터 | ~18MB |
| `public/worker/` | Piper 내부 Web Worker JS | ~60MB |

## URL 처리 (Electron / 웹 자동 감지)

`src/lib/api.ts`에서 환경을 자동 감지합니다:

```typescript
// Electron 프로덕션: file://로 로드되므로 백엔드 주소를 127.0.0.1:8000으로 고정
// ('localhost'는 Windows에서 IPv6(::1)로 풀려 uvicorn IPv4 전용 서버에 닿지 않을 수 있음)
// Electron dev 모드는 Vite dev 서버에서 로드되므로 프록시 경유
export const API_BASE = isElectron && !isViteDev
  ? 'http://127.0.0.1:8000' // Electron 프로덕션: 직접 백엔드 호출
  : ''                      // 그 외: Vite 프록시(/api, /slides, /health 등) 경유

export const WS_BASE = isElectron && !isViteDev
  ? 'ws://127.0.0.1:8000'
  : `${window.location.protocol === 'https:' ? 'wss:' : 'ws:'}//${window.location.host}`

export const WS_PIPELINE_URL = `${WS_BASE}/ws/pipeline`
```

웹 개발 환경에서 백엔드 주소는 `BACKEND_URL` 환경변수로 제어합니다 (기본값: `http://localhost:8000`). 자세한 내용은 [환경설정](../setting/환경설정.md)을 참고하세요.
