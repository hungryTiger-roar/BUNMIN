# 백엔드 설계 (MVP)

## 개요

강의 시스템 전체가 아닌, **화면 공유 상태에서 번역 알고리즘 동작**을 최우선 목표로 함.

- 인증, 강의실 관리, DB 등은 MVP 이후
- 명세서 상 핵심 기능 우선 구현

---

## API 명세

### 슬라이드 사전 처리

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/slides/upload` | PDF 업로드 → 백그라운드 OCR+번역 |
| GET | `/slides/status/{id}` | 처리 상태 확인 |
| GET | `/slides/pages/{id}` | 번역된 페이지 데이터 |

### WebSocket 실시간

| 경로 | 설명 |
|------|------|
| `/ws/pipeline` | 통합 실시간 처리 |

### 헬스체크

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/health` | 서버 상태 확인 |

## WebSocket 메시지 형식

### 강의자 → 서버

```json
// 음성 전송
{
  "type": "audio",
  "audio": "<base64 PCM>",
  "sample_rate": 16000
}

// 화면 전송
{
  "type": "screen",
  "image": "<base64 JPEG>",
  "slide_id": "abc123"
}
```

### 서버 → 수강자

```json
// 번역 결과 (음성 + 자막)
{
  "type": "transcription",
  "original": "한국어 텍스트",
  "translated": "English text",
  "audio": "<base64 WAV>"
}

// 화면 오버레이
{
  "type": "overlay",
  "items": [
    {
      "original": "제목",
      "translated": "Title",
      "bbox": [x1, y1, x2, y2]
    }
  ]
}
```

```

│   ├── routers/

│

├── docker-compose.yml
└── requirements.txt
```

## AI 파이프라인

### 음성 번역 (실시간)

```
음성 입력 → ASR → MT → TTS → 번역 음성 출력
            (한→텍) (한→영)  (영→음성)
```

### 화면 번역 (사전 처리 + 매칭)

```
[강의 전]
PDF 업로드 → 페이지별 OCR → MT → 저장

[강의 중]
화면 캡처 → 해시 매칭 → 저장된 번역 반환
```

---

## 슬라이드 사전 처리 흐름

```
1. POST /slides/upload (PDF 파일)
       ↓
2. 백그라운드 처리 (각 페이지):
   - PDF → 이미지 변환
   - OCR: 텍스트 + 좌표(bbox) 추출
   - MT: 한국어 → 영어 번역
   - 해시 생성 (매칭용)
       ↓
3. GET /slides/status/{id}
   → {"status": "completed", "total_pages": 10}
       ↓
4. 실시간 강의:
   화면 캡처 → 해시 매칭 → 저장된 번역 반환 (OCR 생략)
```

## 저장 방식 (MVP)

| 항목 | 저장 방식 |
|------|-----------|
| 슬라이드 데이터 | 메모리 (Python dict) |
| 번역 결과 | 실시간 전송만 (저장 X) |

**MVP에서는 DB 없이 메모리로 진행**

## 가정 사항

| 항목 | 가정 |
|------|------|
| 동시 강의 | 1개 (MVP) |
| 파일 형식 | PDF만 허용 |
| 확대/축소 | 안 함 |
| 판서/필기 | 안 함 |
| 애니메이션 | 없음 (PDF) |
| 다국어 | 한→영만 (MVP) |
| 동기화 | ~3초 지연 허용 |
