# 백엔드 설계

## 개요

FastAPI 기반 AI 백엔드 서버.
웹 개발 환경에서는 독립 실행되며, Electron 배포 시 PyInstaller로 단일 exe에 번들링됩니다.

---

## 기술 스택

| 분류 | 기술 |
|------|------|
| Web Framework | FastAPI + uvicorn |
| 실시간 통신 | WebSocket |
| AI - ASR | faster-whisper / CTranslate2 (models/whisper-large-v3-turbo-ct2-int8) |
| AI - NMT (실시간) | CTranslate2 CT2 (facebook/nllb-200-distilled-600M, int8) → HF transformers 폴백 |
| AI - 슬라이드 번역 | Qwen3-VL-4B (VLM 기반, VRAM 자동 감지로 4bit/8bit 양자화) |
| AI - OCR | Surya OCR (Transformer 기반, GPU 최적화) |
| PDF 처리 | PyMuPDF |
| 빌드 | PyInstaller |

---

## API 명세

### 슬라이드

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/slides/upload` | PDF 업로드 → 백그라운드 VLM 번역 시작 |
| GET | `/slides/list` | 업로드된 강의자료 전체 목록 (수강자 번역본 다운로드용) |
| GET | `/slides/status/{id}` | 처리 상태 확인 (pending/processing/completed/failed) |
| GET | `/slides/pages/{id}` | 번역된 페이지 전체 데이터 |
| GET | `/slides/image/{id}/{page}` | 슬라이드 페이지 이미지 반환 (`?translated=true`로 번역본) |
| GET | `/slides/download/{id}` | PDF 다운로드 (`?type=original` 또는 `?type=translated`) |

### 자막

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/transcripts` | 저장된 강의 세션 목록 (started_at 내림차순) |
| GET | `/transcripts/{session_id}` | 세션 자막 전체 JSON 조회 (종료된 세션만) |
| GET | `/transcripts/{session_id}/download` | 자막 다운로드 (`?format=json\|srt\|txt`) |
| DELETE | `/transcripts/{session_id}` | 자막 삭제 |

### 네트워크

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/network/info` | 강의자 PC의 LAN IP + 수강자 초대 URL 반환 |

### 모드 전환

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/mode/current` | 현재 모드 + 로드된 모델 목록 확인 |
| POST | `/api/mode/slide` | 슬라이드 번역 모드로 전환 (실시간 모델 언로드) |
| POST | `/api/mode/realtime` | 실시간 번역 모드로 전환 (VLM 언로드 → ASR/NMT-ASR 로드. OCR은 슬라이드 전용이라 재로드 안 함) |

### WebSocket

| 경로 | 설명 |
|------|------|
| `/ws/pipeline` | 강의자/수강자 통합 실시간 처리 |

### 헬스체크

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET / HEAD | `/health` | 모델 로딩 상태 및 진행률 확인 |
| GET(thread) | `port 18765` | GIL 무관 전용 health 서버 |

## WebSocket 메시지 형식

### 클라이언트 → 서버

```json
// 역할 등록 (첫 메시지, 필수)
{ "type": "register", "role": "lecturer", "name": "강의자이름" }
{ "type": "register", "role": "student", "name": "수강자이름" }

// 음성 전송 (강의자)
{ "type": "audio", "audio": "<base64 WAV>", "sample_rate": 16000, "sentAt": 1710000000000 }

// 화면 공유 프레임 (강의자)
{ "type": "screen", "data": "<base64 JPEG>" }

// 슬라이드 선택 (강의자)
{ "type": "slide_select", "slide_id": "abc123" }

// 페이지 변경 (강의자)
{ "type": "page_change", "slide_id": "abc123", "page": 3 }

// 강의 제어 (강의자)
{ "type": "lecture_start", "slide_id": "abc123", "mode": "slide" }
{ "type": "lecture_pause" }
{ "type": "lecture_resume" }
{ "type": "lecture_end" }

// 발표 모드 변경 (강의자)
{ "type": "presentation_mode", "mode": "slide" }

// 커서 위치 (강의자, 20fps)
{ "type": "cursor", "x": 0.5, "y": 0.3, "visible": true, "color": "#60A5FA" }

// 강의 제목 설정 (강의자)
{ "type": "lecture_title", "title": "강의 제목" }

// 강의자 이름 변경 (강의자)
{ "type": "lecturer_name", "name": "새 이름" }

// 채팅 메시지 (강의자/수강자 공통)
{ "type": "chat_message", "text": "채팅 내용" }

// 참여자 목록 요청 (강의자/수강자)
{ "type": "participants_request" }

// WebRTC 시그널링 — 강의자 송신 (수강자별 offer / ICE)
//   서버는 단순 relay (target 수강자에게 forward).
{ "type": "webrtc_offer", "target": "student-uuid", "sdp": { ... } }
{ "type": "webrtc_ice",   "target": "student-uuid", "candidate": { ... } }

// WebRTC 시그널링 — 수강자 송신 (강의자에게 answer / ICE)
//   서버가 sender 필드 (수강자 id) 를 추가해 강의자에게 forward.
{ "type": "webrtc_answer", "sdp": { ... } }
{ "type": "webrtc_ice",    "candidate": { ... } }

// heartbeat 응답 (강의자/수강자 공통)
{ "type": "pong" }
```

### 서버 → 클라이언트

```json
// 역할 등록 확인 — 강의자
{ "type": "registered", "role": "lecturer" }

// 역할 등록 확인 — 수강자 (id, name 추가)
{ "type": "registered", "role": "student", "id": "uuid", "name": "수강자이름" }

// 강의 세션 시작 확인 (강의자에게만)
{ "type": "session_started", "session_id": "a1b2c3d4" }

// 번역 결과 (수강자에게 브로드캐스트, 강의자에게도 전송)
// TTS는 수강자 브라우저 WASM이 처리 — 서버는 텍스트만 전송
//   speechStartAt: 강사 wall-clock 의 발화 시작 시각 (학생측 wall-clock + delay 스케줄 anchor)
//   sentAt: chunk 송출 시각 (≈ 발화 종료 시각). KSS sub-sentence 분리 시 sub_* 우선
//   asrMs: ASR 처리 소요 시간 — 학생측 latency 진단용
{
  "type": "transcription",
  "seq": 1,
  "original": "한국어 텍스트",
  "translated": "English text",
  "speechStartAt": 1710000000000,
  "sentAt": 1710000003000,
  "asrMs": 250
}

// 화면 공유 프레임 (수강자)
{ "type": "screen", "image": "<base64 JPEG>", "slide_id": "abc123", "page": 3 }

// 강의 제어 동기화 (수강자)
{ "type": "lecture_start", "slide_id": "abc123", "session_id": "a1b2c3d4" }
{ "type": "lecture_pause" }
{ "type": "lecture_resume" }
{ "type": "lecture_end", "session_id": "a1b2c3d4" }
{ "type": "page_change", "page": 3 }
{ "type": "slide_select", "slide_id": "abc123" }
{ "type": "presentation_mode", "mode": "screen" }

// 수강자 접속 수 (접속/해제 시 갱신)
{ "type": "student_count", "count": 5 }

// 참여자 목록
{
  "type": "participants",
  "lecturer": { "name": "강의자이름", "connected": true },
  "students": [{ "id": "uuid", "name": "수강자이름" }]
}

// 강의 제목
{ "type": "lecture_title", "title": "강의 제목" }

// 커서 위치 (수강자에게 브로드캐스트)
{ "type": "cursor", "x": 0.5, "y": 0.3, "visible": true, "color": "#60A5FA" }

// 슬라이드 번역 오버레이 (수강자 — 슬라이드 모드에서 화면공유 프레임과 함께 전송)
{
  "type": "overlay",
  "items": [
    { "original": "한국어 텍스트", "translated": "English text", "bbox": [x1, y1, x2, y2], "confidence": 0.95 }
  ]
}

// 채팅 메시지 브로드캐스트 — 강의자 발신
{
  "type": "chat_message",
  "id": "uuid",
  "sender": "lecturer",
  "name": "강의자이름",
  "text": "메시지 내용",
  "timestamp": 1710000000000
}

// 채팅 메시지 브로드캐스트 — 수강자 발신 (student_id 포함)
{
  "type": "chat_message",
  "id": "uuid",
  "sender": "student",
  "name": "수강자이름",
  "student_id": "uuid",
  "text": "메시지 내용",
  "timestamp": 1710000000000
}

// WebRTC 시그널링 — 강사 → 수강자 (1:N relay, target 수강자에게만)
{ "type": "webrtc_offer", "sdp": { ... } }
{ "type": "webrtc_ice",   "candidate": { ... } }

// WebRTC 시그널링 — 수강자 → 강사 (sender 필드로 어느 수강자 응답인지 식별)
{ "type": "webrtc_answer", "sender": "student-uuid", "sdp": { ... } }
{ "type": "webrtc_ice",    "sender": "student-uuid", "candidate": { ... } }

// heartbeat (서버 → 클라이언트, 20초 주기)
{ "type": "ping" }
```

## AI 파이프라인

### 음성 번역 (실시간)

```
강사 마이크 입력
   │
   ├── (a) WebSocket → 백엔드 ASR → NMT → "transcription" 메시지
   │        (한→텍)    (한→영)              ↓
   │                                  수강자 브라우저 WASM TTS
   │                                  (piper-tts-web, 클라이언트 로컬 합성)
   │
   └── (b) WebRTC peer-to-peer → 수강자 브라우저 DelayNode (15s)
            (강사 원본 음성 직접 송출)        ↓
                                       수강자 audioLang='original' 선택 시 출력
```

- 경로 (a) — 텍스트만 전송 (서버 네트워크 부하 없음). 수강자가 `audioLang='en'` 일 때 piper-tts-web 으로 합성.
- 경로 (b) — WebSocket 은 시그널링만 (offer/answer/ICE relay), 실 음성은 P2P. 수강자가 `audioLang='original'` 일 때 출력.
- 두 경로는 학생측 동일한 wall-clock + 15s lag 으로 정렬되어 자막 / 그림 / 음성이 모두 sync. 자세한 sync 모델은 [프론트엔드 설계](프론트_설계.md#수강자-음성-출력--이중-파이프라인) 참고.

### 슬라이드 번역

```
[강의 전 — PDF 업로드 시]
PDF 업로드
  → 페이지별 PNG 이미지 변환 (PyMuPDF)
  → Surya OCR (텍스트 영역 감지, Transformer 기반)
  → Qwen3-VL-4B (한→영 번역, 번역 이미지 합성)
  → 번역 이미지 저장 + overlay_items 생성 (원문/번역/bbox)

[강의 중 — 화면공유 프레임 수신 시]
화면공유 프레임 → 수강자 전달
                 ↘ 슬라이드 모드 시: 사전 처리된 overlay_items 전송
```

실시간 OCR+번역 없음 — 강의 중 슬라이드 처리는 모두 사전 저장된 데이터를 반환합니다.

## 모델 로딩 순서

앱 시작 시 2단계로 로딩 (로딩 화면에 진행률 표시):

**1단계: 병렬 다운로드** — 미캐시 모델만 병렬로 HuggingFace 다운로드

**2단계: 순차 메모리 로딩**
1. ASR (models/whisper-large-v3-turbo-ct2-int8) — 1/3
2. NMT-ASR (facebook/nllb-200-distilled-600M) — 2/3
3. OCR (Surya OCR) — 3/3

슬라이드 번역(Qwen3-VL-4B)은 PDF 업로드 시점에 온디맨드로 로드됩니다.

`SKIP_STARTUP_MODELS=true` 설정 시 ASR/NMT/OCR을 건너뛰고 즉시 준비 상태로 전환됩니다 (슬라이드 번역 전용 모드).

로딩 상태는 `/health` 엔드포인트 및 `__AUNION_STATUS__` stdout 신호로 Electron에 전달됩니다.
port 18765 전용 스레드 서버는 GIL에 무관하게 항상 응답합니다.

## 저장 방식

| 항목 | 저장 방식 |
|------|-----------|
| 슬라이드 데이터 | 메모리 (Python dict) + 파일시스템 |
| 슬라이드 PDF | `uploads/slides/{slide_id}.pdf` |
| 슬라이드 이미지 (원본) | `uploads/images/{slide_id}_{page}.png` |
| 슬라이드 이미지 (번역본) | `uploads/translated/{slide_id}_{page}.png` |
| 번역 PDF | `uploads/translated/{slide_id}_translated.pdf` |
| 강의 자막 | `uploads/transcripts/{session_id}.jsonl / .json / .meta.json` |
| 모델 캐시 | 개발: `backend/cache/huggingface` / 배포: `%LOCALAPPDATA%\Aunion AI\cache\huggingface` |
| 에러 로그 | `%LOCALAPPDATA%\Aunion AI\error_log.txt` |

> `uploads/` 경로는 각 라우터 파일 위치 기준으로 계산한 **절대 경로**로 항상 프로젝트 루트에 생성됩니다 (`Path(__file__).parent × 4` 역산).
