# 백엔드 설계

## 개요

FastAPI 기반 AI 백엔드 서버.
웹 개발 환경에서는 독립 실행되며, Electron 배포 시 PyInstaller로 단일 exe에 번들링됩니다.

---

## 기술 스택

| 분류 | 기술 |
|------|------|
| Web Framework | FastAPI + uvicorn |
| 실시간 통신 | WebSocket |
| AI - ASR | transformers (CohereLabs/cohere-transcribe-03-2026) |
| AI - NMT (실시간) | transformers (facebook/nllb-200-distilled-1.3B) |
| AI - 슬라이드 번역 | Qwen3-VL-8B + LoRA (VLM 기반, EasyOCR + 번역 통합) |
| AI - TTS | Piper TTS (en_US-lessac-medium.onnx) |
| AI - OCR | RapidOCR (ONNX 기반, CPU 최적화) |
| PDF 처리 | PyMuPDF |
| 빌드 | PyInstaller |

## API 명세

### 슬라이드

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/slides/upload` | PDF 업로드 → 백그라운드 OCR+번역 |
| GET | `/slides/status/{id}` | 처리 상태 확인 |
| GET | `/slides/pages/{id}` | 번역된 페이지 전체 데이터 |
| GET | `/slides/image/{id}/{page}` | 슬라이드 페이지 이미지 반환 |
| GET | `/slides/download/{id}` | PDF 다운로드 (원본/번역 선택) |

### WebSocket

| 경로 | 설명 |
|------|------|
| `/ws/pipeline` | 강의자/수강자 통합 실시간 처리 |

### 헬스체크

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET / HEAD | `/health` | 모델 로딩 상태 및 진행률 확인 |
| GET(thread) | `port 18765` | GIL 무관 전용 health 서버 |

## WebSocket 메시지 형식

### 클라이언트 → 서버 (강의자)

```json
// 역할 등록
{ "type": "register", "role": "lecturer" }

// 음성 전송
{ "type": "audio", "audio": "<base64 PCM>", "sample_rate": 16000 }

// 화면 공유 프레임
{ "type": "screen", "data": "<base64 JPEG>" }

// 슬라이드 선택
{ "type": "slide_select", "slide_id": "abc123" }

// 페이지 변경
{ "type": "page_change", "slide_id": "abc123", "page": 3 }

// 강의 제어
{ "type": "lecture_start", "slide_id": "abc123", "mode": "slide" }
{ "type": "lecture_pause" }
{ "type": "lecture_resume" }
{ "type": "lecture_end", "slide_id": "abc123" }

// 발표 모드 변경
{ "type": "presentation_mode", "mode": "slide" }
```

### 서버 → 클라이언트 (수강자)

```json
// 번역 결과 (자막 + 음성)
{
  "type": "transcription",
  "original": "한국어 텍스트",
  "translated": "English text",
  "audio": "<base64 WAV>"
}

// 화면 공유 프레임
{ "type": "screen", "image": "<base64 JPEG>" }

// 강의 제어 동기화
{ "type": "lecture_start", "slide_id": "abc123" }
{ "type": "lecture_pause" }
{ "type": "lecture_resume" }
{ "type": "lecture_end" }
{ "type": "page_change", "page": 3 }
{ "type": "presentation_mode", "mode": "screen" }
```

## AI 파이프라인

### 음성 번역 (실시간)

```
마이크 입력 → ASR → NMT → TTS → 번역 음성 + 자막 출력
             (한→텍) (한→영) (영→음성)
```

### 슬라이드 번역

```
[강의 전]
PDF 업로드 → 페이지별 이미지 변환 → EasyOCR → Qwen3-VL(LoRA) → 번역 이미지 저장

[강의 중]
슬라이드 페이지 번호 → 저장된 번역 이미지 반환 (실시간 처리 없음)
```

## 모델 로딩 순서

앱 시작 시 2단계로 로딩 (로딩 화면에 진행률 표시):

**1단계: 병렬 다운로드** — 미캐시 모델만 병렬로 HuggingFace 다운로드

**2단계: 순차 메모리 로딩**
1. ASR (Cohere ASR) — 1/4
2. NMT-ASR (facebook/nllb-200-distilled-1.3B) — 2/4
3. TTS (Piper TTS) — 3/4
4. OCR (RapidOCR) — 4/4

슬라이드 번역(Qwen3-VL)은 PDF 업로드 시점에 온디맨드로 로드됩니다.

로딩 상태는 `/health` 엔드포인트 및 `__AUNION_STATUS__` stdout 신호로 Electron에 전달됩니다.
port 18765 전용 스레드 서버는 GIL에 무관하게 항상 응답합니다.

## 저장 방식

| 항목 | 저장 방식 |
|------|-----------|
| 슬라이드 데이터 | 메모리 (Python dict) + 파일시스템 |
| 모델 캐시 | 개발: `backend/cache/huggingface` / 배포: `%LOCALAPPDATA%\Aunion AI\cache\huggingface` |
| 슬라이드 파일 | `%LOCALAPPDATA%\Aunion AI\slides` |
| 에러 로그 | `%LOCALAPPDATA%\Aunion AI\error_log.txt` |
