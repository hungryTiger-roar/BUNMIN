데이터 구조 / DB 덤프 (대체)

본 프로젝트는 RDBMS 를 사용하지 않습니다. 일반적인 DB 덤프 파일은 해당 없음. 대신 본 문서가 데이터 저장 구조와 ERD 대체 자료를 제공합니다.

목차

  1. 왜 RDBMS 를 안 썼나
  2. 데이터 저장 위치 (전체 맵)
  3. 데이터 구조 명세 (ERD 대체)
  4. 통신 데이터 (메모리, 영속화 X)
  5. DB 덤프 대체 자료
  6. 백업 / 마이그레이션
  7. 정리 한 줄

1. 왜 RDBMS 를 안 썼나

항목본 프로젝트 데이터 특성
강의 자료 (PDF)파일 자체가 단위 → 파일시스템 이 자연
자막 (강의별)강의 종료 후 텍스트 저장 → 파일시스템 (JSON)
학생 설정 (이름·자막 언어·음성)브라우저 로컬 → LocalStorage / IndexedDB
TTS voice (학생 측)30MB Piper voice → IndexedDB 캐시
AI 모델 (~14GB)대용량 binary → 파일시스템 (%LOCALAPPDATA%)
강의 세션 (실시간)WebRTC + WebSocket 통신 → 메모리 (영속화 X)
사용자 계정없음 (익명 학생 + 강사 권한 분리)

→ 모든 데이터가 "관계 없는 단일 단위" 또는 "브라우저 영속화 대상" 이라 RDBMS 의 트랜잭션 / 조인 / 인덱스가 필요 없음. 단순한 파일/스토리지 모델이 적합.

2. 데이터 저장 위치 (전체 맵)

2.1 강사 노트북 측

%LOCALAPPDATA%\Aunion AI\
│
├─ uploads\                                  ← 강의 자료 (강사 업로드 PDF)
│   ├─ <lecture-id>\
│   │   ├─ original.pdf                      ← 원본 PDF
│   │   ├─ translated.pdf                    ← 번역된 PDF (VLM 결과)
│   │   ├─ pages\                            ← 페이지별 이미지 (slide preview)
│   │   │   ├─ page_001.png
│   │   │   └─ ...
│   │   └─ metadata.json                     ← 강의 메타 (title, page count, created_at)
│   └─ cache\                                ← 슬라이드 번역 중간 결과 (OCR/VLM)
│
├─ transcripts\                              ← 강의 자막 (사후 저장)
│   ├─ <lecture-id>-<timestamp>.json         ← {time, speaker, text_ko, text_en}[]
│   └─ ...
│
├─ cache\
│   └─ huggingface\                          ← HF 모델 캐시 (~14GB)
│       ├─ models--Qwen--Qwen3-VL-4B-Instruct\
│       └─ ...
│
├─ logs\                                     ← 백엔드 로그
├─ error_log.txt                             ← Electron 에러 로그
├─ window-state.json                         ← 창 위치/크기 영속화
└─ .last-run-version                         ← 마지막 실행 버전 (마이그레이션 트리거)

2.2 학생 노트북 측 (브라우저 IndexedDB / LocalStorage)

%APPDATA%\Aunion AI\                         (Electron app.getPath('userData'))
│
├─ IndexedDB\
│   └─ <origin>\                             ← Piper voice (학생 TTS)
│       └─ en_US-lessac-medium.onnx          ← ~30MB
│
├─ Local Storage\                            ← 학생 설정
│   └─ leveldb\                              ← key-value
│       (preferences-store: studentName, audioLang, subtitleStyle, volume)
│
├─ Service Worker\                           ← 일부 정적 자원 캐시
├─ Cache\
└─ Cookies\

4. 통신 데이터 (메모리, 영속화 X)

실시간 통신 데이터는 모두 메모리 + WebSocket / WebRTC 로 흐름. 영속화 안 됨.

4.1 WebSocket 메시지 타입 (강사 ↔ 학생)

타입방향페이로드 요약
registerbothrole (lecturer/student), name
lecture_start / lecture_endL→S강의 시작/종료
pause / resumeL→S일시정지 토글
page_changeL→S페이지 변경
presentation_modeL→S발표 모드 (slide / blackboard)
cursorL→S마우스 위치 (x, y, visible)
draw_begin / draw_point / draw_endL→S판서 좌표
transcriptionL→S한국어 텍스트 + 영어 텍스트 + 타이밍 메타
audioL→SASR 결과 (WAV base64) — 자막 생성 후 폐기
webrtc_offer / webrtc_answer / webrtc_icebothWebRTC 시그널링
student_audio_langS→L학생 측 audioLang 변경 (강사 패널 표시용)
chatboth채팅 메시지

4.2 WebRTC 미디어

  • Audio: 강사 마이크 → 학생 (Opus 48kHz mono, ~32kbps)
  • Video: 강사 화면 공유 → 학생 (강의 시 화면 공유 시만)

5. DB 덤프 대체 자료

일반 RDBMS 덤프 파일이 없는 대신, "참고용 샘플 데이터" 를 다음 위치에서 확인 가능:

자료위치용도
샘플 강의 자료 (PDF)발표자료/ (저장소)시연용 PDF
도메인 용어집 (glossary)glossary/*.csvASR/NMT 정확도 향상용 (한↔영 용어 매핑)
자막 JSON 샘플(강의 1회 실행 후 transcripts/ 생성)구조 확인
AI 모델 다운로드 스크립트scripts/setup.js모델 종류·버전 정의

5.1 첫 실행 데이터 시드

설치 직후 (모든 강의 데이터가 비어있을 때) — npm run setup 또는 인스톨러가 자동 생성:

항목시드 동작
models/HuggingFace 에서 자동 다운로드
glossary/*.csv저장소에 커밋되어 있음 → 그대로 사용
uploads/빈 폴더 (사용자가 PDF 업로드 시 채워짐)
transcripts/빈 폴더 (강의 1회 실행 시 채워짐)

6. 백업 / 마이그레이션

6.1 사용자 데이터 백업

# 강사 노트북 (강의 자료 + 자막 + 모델)
Copy-Item -Recurse "$env:LOCALAPPDATA\Aunion AI" -Destination "D:\backup\Aunion AI"

# 학생 노트북 (학생 설정 + TTS voice)
Copy-Item -Recurse "$env:APPDATA\Aunion AI" -Destination "D:\backup\Aunion AI-student"

6.2 마이그레이션 (다른 PC 로 이전)

항목이전 방법
강의 자료%LOCALAPPDATA%\Aunion AI\uploads\ 폴더 복사
자막%LOCALAPPDATA%\Aunion AI\transcripts\ 폴더 복사
AI 모델 (~14GB)%LOCALAPPDATA%\Aunion AI\cache\huggingface\ 복사 (재다운로드 회피)
학생 설정학생이 직접 재설정 (LocalStorage 이전 어려움)

6.3 완전 초기화

Remove-Item -Recurse "$env:LOCALAPPDATA\Aunion AI"
Remove-Item -Recurse "$env:APPDATA\Aunion AI"

→ 첫 실행 상태로 복귀. 인스톨러 재실행 시 다시 모델 다운로드.

7. 정리 한 줄

본 프로젝트는 RDBMS 미사용. 모든 데이터가 "파일시스템 + 브라우저 영속화" 모델이라 DB 덤프 파일이 의미 없습니다. 본 문서가 ERD 대체 자료입니다.
