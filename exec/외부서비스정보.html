1. HuggingFace Hub (모델 다운로드)

1.1 용도

AI 모델 파일 다운로드. 한 번 받으면 로컬 캐시 → 이후 오프라인 동작.

1.3 가입 / 인증 필요?

기본 사용은 가입 불필요 (모두 공개 모델). 단, 다음 케이스에서만 토큰 필요:

시나리오HF_TOKEN 필요
위 모델만 다운로드아니오
다운로드 속도 제한 회피 (rate limit)예 (선택)
Gated 모델 추가 시예

1.4 토큰 생성 방법 (필요 시)

  1. https://huggingface.co/ 가입
  2. Settings → Access Tokens → New token (type: read)
  3. .env 의 HF_TOKEN=hf_xxx... 에 붙여넣기

1.5 다운로드 자동화

npm run setup 의 6단계에서 scripts/setup.js 가 자동 호출:

  • HuggingFace Hub Python SDK 로 모델 받음
  • models/ 폴더에 저장 (gitignored, 사용자별)
  • CTranslate2 변환 (Whisper, NLLB) → 양자화로 메모리 절약

1.6 캐시 위치

환경위치
개발 모드models/, ~/.cache/huggingface/
설치본%LOCALAPPDATA%\Aunion AI\cache\huggingface\

1.7 URL 차단 환경 대응

회사/학교 방화벽에서 huggingface.co 차단 시:

  • 다른 PC 에서 다운로드 후 models/ 폴더 통째 복사
  • 또는 사내 미러 서버 운영 → 환경 변수로 redirect

2. Google STUN 서버 (WebRTC NAT traversal)

2.1 용도

WebRTC P2P 연결 시 NAT 뒤의 학생/강사 PC 의 공인 IP 발견.

2.2 사용 주소

stun:stun.l.google.com:19302

코드 위치: frontend/src/pages/Student.tsx:312, frontend/src/pages/Lecturer.tsx:316

2.3 가입 / 인증

불필요 — Google 공개 STUN 서버. 토큰 / 계정 없음.

2.4 대안 (Google 차단 환경)

STUN 서버주소
Cloudflarestun:stun.cloudflare.com:3478
자체 호스팅 (coturn)별도 서버 구축 필요

코드에서 iceServers 항목 추가 / 교체로 변경 가능.

3. 자체 TURN 서버 (강사 노트북)

3.1 용도

WebRTC P2P 가 차단된 환경 (사내망 등) 에서 강사 노트북이 학생들의 트래픽을 relay.

3.2 구현

항목값
라이브러리node-turn 0.0.6 (npm)
코드turn-server/index.js
포트47878
인증 usernameaunion
인증 credentialaunion-secret
실행Electron 앱 시작 시 자동 spawn

3.3 클라이언트 측 설정

// Lecturer.tsx:317-318
{ urls: `turn:${turnHost}:47878`, username: 'aunion', credential: 'aunion-secret' },
{ urls: `turn:${turnHost}:47878?transport=tcp`, ... }

turnHost 는 강사 노트북 LAN IP (getLanIp() 로 자동 감지, 백엔드 /network/info 엔드포인트).

3.4 보안

  • 인증 정보가 코드에 하드코딩 (LAN 환경 사용 가정)
  • 외부 공개 서비스 아님 — 같은 LAN 내 학생만 접근
  • 강사 노트북 종료 시 함께 종료

7. 외부 서비스를 사용하지 않은 영역 (의도적 설계)

본 프로젝트는 다음을 의도적으로 외부 의존 없이 구현했습니다:

기능외부 SaaS 대신이유
ASROpenAI Whisper / Azure Speech 대신 로컬 Whisper비용 0, 오프라인, 개인정보 보호
NMTDeepL / Google Translate 대신 로컬 NLLB동일
TTSGoogle TTS / Azure TTS 대신 학생 측 WebAssembly Piper서버 부하 0, 학생 분산 처리, sync 정확도
화면 공유Zoom SDK 대신 자체 WebRTC비용 0, 강의 흐름 맞춤 sync
인증OAuth / Firebase Auth 대신 익명 학생 + 강사 권한 분리학원·회의실 환경 가정
로깅Sentry / DataDog 대신 로컬 파일 (error_log.txt)단순함, 개인정보 보호

결과: 클라우드 비용 0, SaaS 가입 0, 학원·회의실·교실 환경에 강사 노트북 1대로 즉시 배포 가능.

8.2 인터넷 미접근 환경 운용 가능?

가능. 단 다음 조건:

  1. 다른 PC 에서 npm run setup 실행해 models/ 폴더 채워서 복사
  2. 첫 실행 후 VLM 다운로드 완료 상태로 PC 복사 (%LOCALAPPDATA%\Aunion AI\cache\huggingface\)
  3. WebRTC 의 STUN 의존 — 같은 LAN 내 P2P 면 STUN 도 불필요 (host candidate 만으로 연결)
