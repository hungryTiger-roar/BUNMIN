0. 본 프로젝트의 핵심 특이사항 (먼저 읽기)

항목본 프로젝트의 선택
백엔드 형태Python FastAPI (PyInstaller 로 패키징, 별도 WAS / 톰캣 없음)
프론트엔드 형태Vite + React (Electron 앱으로 packaged, 별도 웹서버 없음)

외부 서버자체 호스팅 (강사 노트북) — 클라우드 서버 미사용
통신WebRTC P2P + WebSocket (TURN 서버 자체 호스팅)

JVM / WAS / DB 가 일반적인 웹 프로젝트와 달라 본 매뉴얼은 "사용한 기술 스택 그대로" 기재합니다.

8. 배포 시 특이사항

8.1 설치 위치

%LOCALAPPDATA%\Aunion AI\          ← per-user 설치, UAC 권한 X

8.2 재설치 동작

항목처리
Aunion AI.exe, backend\, app.asar덮어씌움
모델 캐시 / 강의자료 / 자막 / 학생 설정보존
AppId (installer.iss:8)동일 → "업데이트" 로 인식

8.3 첫 실행 시 추가 다운로드

  • 슬라이드 번역용 VLM (Qwen3-VL-4B-Instruct, 약 14GB) 미캐시 시 HuggingFace 에서 자동 다운로드
  • 디스크 35GB 여유 권장 (본체 3.4GB + VLM 14GB + symlink copy 사본 14GB + 마진)
  • 설치 마법사에서 디스크 여유 자동 체크 → 부족 시 경고

8.4 Windows 보안 정책 호환

  • PrivilegesRequired=lowest + asInvoker manifest → UAC 안 뜸
  • WinError 448 (신뢰할 수 없는 탑재 지점) 우회 — HF symlink → copy 모드 강제
  • 백엔드 grace period 5분 — Electron 종료 후에도 학생 자막 다운로드 처리

10. 트러블슈팅

npm run setup 도중 실패

conda env remove -n aunion -y
npm run setup

PyInstaller 가 frontend/dist 없다고 빌드 중단

npm run build --prefix frontend
npm run build:installer

설치본이 마이크 / 카메라 못 잡음

  • http://127.0.0.1:48000 로딩이 secure context 인정 조건
  • %LOCALAPPDATA%\Aunion AI\error_log.txt 확인

[WinError 448] 신뢰할 수 없는 탑재 지점

Remove-Item -Recurse -Force "$env:LOCALAPPDATA\Aunion AI\cache"
# 앱 재시작 → 자동 재다운로드

Inno Setup 컴파일 실패 (ISCC.exe 못 찾음)

& "$env:LOCALAPPDATA\Programs\Inno Setup 6\ISCC.exe" installer.iss

강사 발화가 학생에게 안 들림

콘솔(F12) 에서 다음 로그 확인:

  • [Lecturer] 초기 replaceTrack → audio=null|<id>
  • [Lecturer] swap → audio=<id> renegotiate=true
  • [Track] audio onunmute → RTP 흐름 시작

학생 측 [Stats] inbound audio packets= 가 증가하면 정상. 0이면 WebRTC 재협상 / TURN 점검.

새로고침 후 영어 TTS 안 들림 (v0.1.0 fix 완료)

Student.tsx:847 의 TTS GainNode 동기화 effect deps 에 isAudioUnlocked 추가됨. v0.1.0 새 빌드 사용 시 정상 작동. 옛 빌드 사용 중이면 콘솔에 Student-CIiFaLnG.js 가 보여야 함 (옛 빌드는 다른 hash).
